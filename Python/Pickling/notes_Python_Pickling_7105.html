Notes: Python- Pickling

Pickling

Pickling in Python refers to the process of converting a Python object into a byte stream, which can later be deserialized (unpickled) to reconstruct the original object. This is useful for saving complex objects like lists, dictionaries, or even custom classes to a file or transferring them over a network.

What is Pickling?

Pickling is the process of converting a Python object into a byte stream so that it can be stored in a file or sent over a network. This is useful for saving program states, caching data, or sharing objects between programs.

Python provides the pickle module for this purpose.


​​​​​​​What is Unpickling?

Unpickling is the reverse of pickling. It involves loading the byte stream and converting it back into a Python object.

​

Before diving into Python Pickle, it’s essential to understand the importance of object serialization. Instead of simply saving data structures in a text file, serialization allows efficient storage and retrieval of complex objects.

Functions in pickle Module

FunctionDescription
pickle.dump(obj, file)Serializes obj and writes it to file
pickle.dumps(obj)Serializes obj and returns a byte stream
pickle.load(file)Reads from file and deserializes the byte stream back into a Python object
pickle.loads(byte_data)Deserializes byte_data back into a Python object

Advantages

Advantages of Using Pickle in Python

  1. Serialization of Complex Objects
    Supports lists, dictionaries, sets, tuples, and even custom objects.
  2. Efficient for Large Data
    Faster than other text-based serialization formats like JSON when working with complex objects.
  3. Preserves Python Objects
    Maintains the structure and types of objects when deserializing.
  4. Easy to Use
    ​​​​​​​The pickle module provides simple dump() and load() functions for saving and loading objects.
Disadvatages

Disadvantages of Pickling

  1. Not Secure
    • Pickle files can execute arbitrary code, leading to security vulnerabilities if loading untrusted data.
  2. Python-Specific Format
    • Pickled objects can only be unpickled in Python, making it non-portable across other programming languages.
  3. Version Compatibility Issues
    • Pickle files may not be compatible between different Python versions.
  4. Larger File Size
    • Pickled files are typically larger than JSON or CSV because they store type information.
Pickling without a File
We can pickle an object into a bytes object instead of a file using pickle.dumps() and retrieve it using pickle.loads().
​​​​​​​
Code:Output:
import pickle

# Sample dictionary
data = {"name": "Alice", "age": 25, "city": "New York"}

# Pickling without a file (converting to bytes)
pickled_data = pickle.dumps(data)

# Unpickling (loading back)
unpickled_data = pickle.loads(pickled_data)

print(unpickled_data)  
# Output: {'name': 'Alice', 'age': 25, 'city': 'New York'}

{'name': 'Alice', 'age': 25, 'city': 'New York'}
Pickling with a File
To store a pickled object in a file, we use pickle.dump() and retrieve it using pickle.load().
Code:Output:
import pickle

# Sample data
data = {"name": "Alice", "age": 25, "city": "New York"}

# Writing to a file
with open("data.pkl", "wb") as file:
    pickle.dump(data, file)

# Reading from the file
with open("data.pkl", "rb") as file:
    loaded_data = pickle.load(file)

print(loaded_data)  
# Output: {'name': 'Alice', 'age': 25, 'city': 'New York'}

{'name': 'Alice', 'age': 25, 'city': 'New York'}
