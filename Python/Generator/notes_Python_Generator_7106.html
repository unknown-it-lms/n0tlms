Notes: Python- Generator

Generator
Introduction
  • Generators are special types of iterators in Python that allow on-the-fly generation of values.
  • They use the yield keyword instead of return.
  •  This allows the function to generate values and pause its execution after each yield, maintaining its state between iterations.
  • They are memory-efficient as they do not store the entire sequence in memory.
Generator Function
Generators in Python are a special type of function that act as iterators, allowing you to iterate over a sequence of values one at a time. They are particularly useful for handling large datasets or infinite sequences efficiently, as they generate values on the fly instead of storing them all in memory at once
Differences Between Generators and Normal Functions
FeatureNormal Function (return)Generator (yield)
Memory UsageStores all values in memoryGenerates values on the fly
ExecutionRuns once and returns a valueCan pause and resume execution
PerformanceCan be slow for large data setsMore efficient for large data
Return TypeReturns a single valueReturns an iterator

Advantages of Generators
  1. Memory Efficient – Suitable for large datasets.
  2. Lazy Evaluation – Generates values only when needed.
  3. Simplifies Code – No need to manage state explicitly.
Use Cases of Generators
  • Reading large files line-by-line (yield instead of storing all lines).
  • Streaming data processing (e.g., sensor data, logs).
  • Generating infinite sequences (e.g., Fibonacci, prime numbers).
  • Implementing pipelines in data processing.
Create a Generator

Create a Generator in Python
Creating a generator in Python is as simple as defining a function with at least one yield statement. When called, this function doesn’t return a single value; instead, it returns a generator object that supports the iterator protocol. The generator has the following syntax in Python:

def generator_function_name(parameters):
    # Your code here
    yield expression
    # Additional code can follow

Example:
In this example, we will create a simple generator that will yield three integers. Then we will print these integers by using Python for loop.

CodeOutput
# A generator function that yields 1 for first time,
# 2 second time and 3 third time
def fun():
    yield 1            
    yield 2            
    yield 3            
 
# Driver code to check above generator function
for val in fun(): 
    print(val)

This generator function fun yields numbers from 1 up to a specified max. Each call to next() on the generator object resumes execution right after the yield statement, where it last left off.

Yield vs Return

Both yield and return are used in functions to send back values, but they behave very differently.

Featurereturnyield
Function TypeRegular functionGenerator function
Memory UsageStores all values in memoryGenerates values on demand
ExecutionTerminates the functionPauses and resumes execution
Return TypeReturns a valueReturns a generator object (iterator)
Multiple CallsExecutes from the beginning every timeResumes from the last yield
return – Used in Regular Functions
  • When return is encountered, the function execution stops.
  • The function returns a single value or a collection (list, tuple, etc.).

Example:

CodeOutput
def square_numbers(n):
    result = []
    for i in range(n):
        result.append(i ** 2)
    return result  # Returns the entire list at once

print(square_numbers(5))		[0, 1, 4, 9, 16]
Downside: If n is large, it consumes a lot of memory since all values are stored at once.
yield – Used in Generator Functions
  • Instead of returning a full list, yield returns one value at a time.
  • The function pauses after yield, allowing iteration without storing everything in memory.

Example:

CodeOutput
def square_numbers(n):
    for i in range(n):
        yield i ** 2  # Generates one value at a time

gen = square_numbers(5)
print(next(gen))  # 0
print(next(gen))  # 1
print(next(gen))  # 4		0
1
4

Advantage: More memory-efficient for large data sets.

When to Use yield vs return

Use return when:

  • You need a single final result.
  • The function should terminate after returning a value.
  • The dataset is small, and memory is not a concern.

Use yield when:

  • You need an iterable sequence without storing it all at once.
  • The function should pause and resume.
  • You are working with large data streams (e.g., reading a file line by line).
Real-World Example: Reading Large Files
CodeOutput
def read_large_file(filename):
    with open(filename, "r") as file:
        for line in file:
            yield line  # Reads one line at a time

for line in read_large_file("big_data.txt"):
    print(line)  # Processes one line at a time

Advantage: Saves memory by reading the file lazily instead of loading everything at once.
Generator Expression
  • A generator expression is similar to list comprehension but uses parentheses () instead of square brackets [].

Generator Expression Syntax

A generator expression has the following syntax,

(expression for item in iterable)
Here, expression is a value that will be returned for each item in the iterable.

The generator expression creates a generator object that produces the values of expression for each item in the iterable, one at a time, when iterated over.

Example

CodeOutput
gen_exp = (x**2 for x in range(5))
print(next(gen_exp))  # 0
print(next(gen_exp))  # 1


# create the generator object
squares_generator = (i * i for i in range(5))
# iterate over the generator and print the values
for i in squares_generator:
    print(i)

Applications of Generators
​​​​​​​Generators are memory-efficient, lazy-evaluated, and useful for handling large datasets. Below are some key applications of generators in real-world scenarios:
​​​​​​​
1. Generating Infinite Sequences
Generators can produce infinite sequences without consuming memory.

2. Processing Large Files Efficiently
Reading large files line by line prevents memory overload.

3. Streaming Data Processing
Used in real-time applications where data arrives continuously (e.g., stock prices, sensor data).

4. Generating Large Data Sets for Testing
Generators are used to create large synthetic datasets for testing algorithms.

5. Implementing Pipelines for Data Processing
Generators can be chained to process data in stages.

6. Memory-efficient List Processing
Instead of storing a large list in memory, generators process elements one by one.

7. Web Scraping with Lazy Loading
Generators help in scraping web pages without loading everything at once.

8. Implementing Custom Iterators
Generators can be used to define custom iterators without needing classes.

9. Database Query Results Streaming
Instead of loading all records into memory, generators fetch data one row at a time.

10. Generating Unique Identifiers
Generators can create unique values dynamically.
