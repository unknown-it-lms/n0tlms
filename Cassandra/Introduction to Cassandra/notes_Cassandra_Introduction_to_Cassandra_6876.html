Introduction to Cassandra
Cassandra is a distributed database system. It is used in many companies for scalability and high availability without compromising performance. Linear scalability and proven fault-tolerance on commodity hardware or cloud infrastructure make it the perfect platform for mission-critical data.

Cassandra is an open-source NoSQL distributed database that manages large amounts of data across commodity servers. It is a decentralized, scalable storage system designed to handle vast volume of data across multiple commodity servers, providing high availability without a single point of failure.
Apache Cassandra is an open-source database that uses a distributed NoSQL architecture for high reliability.
Cassandra can be said as one of the  early NoSQL databases, distributed database based on a hybrid between a tabular store and a key-value store.

Cassandra was created for Facebook but was open-sourced and released to become an Apache project (maintained by the Americal non-profit,  Apache Software Foundation) in 2008. After that, it found top priority in 2010 and is now among the best NoSQL database systems in the world. Cassandra is trusted and used by thousands of companies because of the ease of expansion and  its lack of a single point of failure. 
Currently, it has been deployed to handle databases for Netflix, Twitter, Reddit, etc.   

Apache Cassandra, a NoSQL database, belongs to the big data family of applications and operates as a distributed system, and uses the principle of data partitioning .
