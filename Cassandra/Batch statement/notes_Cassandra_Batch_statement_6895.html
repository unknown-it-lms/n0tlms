Notes: Cassandra- Batch statement

Batch statement

Batch statement is used to combine multiple DML statements to achieve atomicity and isolation when targeting a single partition or only atomicity when targeting multiple partitions. A batch applies all DMLs within a single partition before the data is available, ensuring atomicity and isolation.  

A well constructed batch targeting a single partition can reduce client-server traffic and more efficiently update a table with a single row mutation. A batch can also target multiple partitions, when atomicity and isolation is required. Multi-partition batches may decrease the  throughput and increase the latency.     

 Syntax
BEGIN BATCH
      dml statements;
APPLY BATCH;
eg   
begin batch
      insert into students(roll,name,per)
      values(101,'Sujay',89.00);
      delete 
      from students
      where roll=20;
      update students
      set per=67
      where  roll =23;
apply batch;
