Packages in R

In R programming, packages are collections of functions, data sets, and documentation that extend the capabilities of the base R language. Packages are an essential part of the R ecosystem, and they provide a way to share and distribute code, data, and knowledge among R users.

There are thousands of packages available in R, and they cover a wide range of topics, including statistics, machine learning, data visualization, data manipulation, text mining, and more. Some popular packages in R include:

  1. ggplot2: a package for creating beautiful and informative data visualizations.

  2. dplyr: a package for data manipulation and transformation, including filtering, grouping, and summarizing data.

  3. tidyr: a package for data cleaning and reshaping, including filling missing values and converting data between wide and long formats.

  4. caret: a package for machine learning and predictive modeling, including classification, regression, and clustering.

  5. shiny: a package for creating interactive web applications and dashboards.

Packages in R
  1. Packages in R Programming language are a set of R functions, compiled code, and sample data.
  2. These are stored under a directory called “library” within the R environment.
  3. By default, R installs a group of packages during installation.
  4. Once we start the R console, only the default packages are available by default.
  5. Other packages that are already installed need to be loaded explicitly to be utilized by the R program that’s getting to use them.

What are Repositories?
A repository is a place where packages are located and stored so you can install packages from it.
Organizations and Developers have a local repository, typically they are online and accessible to everyone. Some of the most popular repositories for R packages are:

CRAN:
  1. Comprehensive R Archive Network(CRAN) is the official repository, it is a network of ftp and web servers maintained by the R community around the world.
  2. The R community coordinates it, and for a package to be published in CRAN, the Package needs to pass several tests to ensure that the package is following CRAN policies.
Bioconductor:
  1. Bioconductor is a topic-specific repository, intended for open source software for bioinformatics.
  2. Similar to CRAN, it has its own submission and review processes, and its community is very active having several conferences and meetings per year in order to maintain quality.
Github:
Github is the most popular repository for open source projects. It’s popular as it comes from the unlimited space for open source, the integration with git, a version control software, and its ease to share and collaborate with others.

Install an R-Packages
There are multiple ways to install R Package, some of them are,
Installing Packages From CRAN: For installing Package from CRAN we need the name of the package and use the following command:
install.packages("package name")
Installing Package from CRAN is the most common and easiest way as we just have to use only one command. In order to install more than a package at a time, we just have to write them as a character vector in the first argument of the install.packages() function:

Example:
install.packages(c("vioplot", "MASS"))

Update, Remove and Check Installed Packages in R
To check what packages are installed on your computer, type this command:
installed.packages()

To update all the packages, type this command:
update.packages()

To update a specific package, type this command:
# Update dplyr package
update.packages("dplyr")

How to Load Packages in R Programming Language
When a package is installed, we are ready to use its functionalities.
In R programming, you can load only a particular function from a package using the double colon operator ::. Example:
 packagename::functionname()

Let's say you want to use the mean() function from the stats package, but you don't want to load the entire package. You can use the following code:

stats::mean(x)

This code will only load the mean() function from the stats package and apply it to the object x.

Here's a full example:

# Create a vector
x <- c(1, 2, 3, 4, 5)
# Load only the mean function from the stats package
​​​​​​​stats::mean(x)

This will output the mean of the vector x:
[1] 3

Difference Between a Package and a Library

Unload a Package in R Language
To unload a given package, use the detach() function. The use will be:
detach("package:stats", unload = TRUE)


Working with packages
​​​​​​​In R programming, packages are collections of functions, data sets, and documentation that extend the capabilities of the base R language. To work with packages in R, you can use the following commands and techniques:
  1. Installing packages:
    Use the install.packages() function to install packages from the Comprehensive R Archive Network (CRAN) or other repositories. For example, to install the dplyr package, you can use the following command:
    install.packages("dplyr")

  2. Loading packages:
    Use the library() function or the require() function to load a package into your R session. For example, to load the dplyr package, you can use the following command:
    library(dplyr)

  3. Checking installed packages:
    Use the installed.packages() function to list all installed packages in your R environment. For example, to list all installed packages, you can use the following command:
    installed.packages()
  4. Updating packages:
    Use the update.packages() function to update packages to their latest versions. For example, to update all packages, you can use the following command:
    update.packages()

  5. Searching for packages:
    Use the search() function to list all loaded packages in your R environment. For example, to list all loaded packages, you can use the following command:
    search()

  6. Unloading packages:
    Use the detach() function to unload a package from your R environment. For example, to unload the dplyr package, you can use the following command:
    detach(package:dplyr, unload=TRUE)

  7. Creating packages:
    If you want to create your own package, you can use the devtools package or the usethis package to create a package skeleton, add functions and data sets, and build and publish your package to CRAN or other repositories.


Handling data in R workspace

In R programming, there are various functions and techniques available to handle data in the workspace. Some common approaches are:

  1. Importing data: Use functions such as read.csv(), read.table(), read_excel() and readr::read_csv() to import data from external sources such as CSV files, Excel files, or databases.

  2. Creating data frames: Use the data.frame() function to create a data frame from vectors or lists. Data frames are a fundamental data structure in R, and they allow you to manipulate and analyze data in a tabular format.

  3. Subsetting data: Use the subset() function or the [ ] operator to extract subsets of data based on specific criteria.

  4. Manipulating data: Use functions such as merge(), aggregate(), dplyr::filter(), dplyr::mutate(), dplyr::select(), and tidyr::pivot_longer() to manipulate data in various ways, such as merging datasets, aggregating data, filtering rows, creating new variables, and reshaping data.

  5. Cleaning data: Use functions such as na.omit(), complete.cases(), dplyr::na_if(), and tidyr::fill() to clean and handle missing data, outliers, and other anomalies in the data.

  6. Exporting data: Use functions such as write.csv(), write.table(), write_excel_csv2(), and save() to export data from R to external files or formats.

Reading and importing data from files

In R programming, you can read and import data from a variety of file formats such as CSV, Excel, JSON, TXT, and more. Here are some examples of how to read and import data from different file types:

  1. CSV Files: To read a CSV file, you can use the read.csv() or read.csv2() function, depending on the format of the CSV file. The read.csv() function is used for comma-separated files, while read.csv2() is used for semicolon-separated files. Here's an example:
    # Read a CSV file
    data <- read.csv("filename.csv")

  2. Excel Files: To read an Excel file, you can use the readxl package in R. You need to install the package first using the following command: install.packages("readxl"). Here's an example of how to read an Excel file:
    # Load the readxl library
    library(readxl)

    # Read an Excel file
    data <- read_excel("filename.xlsx")

  3. JSON Files: To read a JSON file, you can use the jsonlite package in R. You need to install the package first using the following command: install.packages("jsonlite"). Here's an example of how to read a JSON file:
    # Load the jsonlite library
    library(jsonlite)
    # Read a JSON file
    data <- fromJSON("filename.json")

  4. TXT Files: To read a TXT file, you can use the read.table() function. Here's an example:
    # Read a TXT file
    data <- read.table("filename.txt", header = TRUE)
Exporting data from r

In R programming, there are several ways to export data from your R environment to external files, including:

  1. CSV file: Use the write.csv() function to export a data frame to a CSV file. For example, if you have a data frame called mydata and you want to export it to a file called mydata.csv, you can use the following command:
    write.csv(mydata, "mydata.csv", row.names = FALSE)

  2. Excel file: Use the openxlsx package to export a data frame to an Excel file. For example, if you have a data frame called mydata and you want to export it to a file called mydata.xlsx, you can use the following commands:
    library(openxlsx)
    write.xlsx(mydata, "mydata.xlsx")

  3. RData file: Use the save() function to save one or more objects in your R environment to an RData file. For example, if you want to save the mydata data frame to a file called mydata.RData, you can use the following command:
    save(mydata, file = "mydata.RData")

  4. Text file: Use the write.table() function to export a data frame or matrix to a text file. For example, if you have a data frame called mydata and you want to export it to a file called mydata.txt, you can use the following command:
    write.table(mydata, "mydata.txt", sep = "\t", row.names = FALSE)
