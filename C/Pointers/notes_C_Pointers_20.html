Notes: C- Pointers

Pointers

Pointers are a fundamental aspect of C programming that provide powerful capabilities by allowing direct memory access and manipulation. Understanding pointers is crucial for achieving efficient and effective memory management in C.

One of the significant benefits of pointers is their ability to optimize program performance. They enable manipulation of arrays and structures without the need for excessive copying, which can save memory and processing time.

Pointers are also essential for dynamic memory allocation, allowing programs to request and release memory as needed, which is vital for the development of large and complex applications.

Moreover, pointers facilitate communication between different parts of a program and functions by passing addresses instead of actual values, enabling the modification of original data.

Need of pointers

Pointers in C programming provide the ability to directly access the memory of your computer. This functionality is crucial because it allows programmers to manipulate the memory locations where data is stored. By using pointers, programmers can perform operations that would be impossible or inefficient with higher-level abstractions.

Pointers are logically needed in some programming tasks, such as swapping the values of two numbers. In C, this may be implemented by passing the addresses of variables to a function that performs the swap. Without pointers, achieving this task would require more complex and cumbersome methods, often involving returning multiple values from functions or re-computing the entire context where values need to be swapped.

When dealing with functions that require large amounts of data, pointers become a valuable tool because they enable efficient data handling. Instead of copying large structures or arrays every time they are passed to a function, a pointer to the data can be passed, reducing the memory footprint and processing overhead. This efficiency is particularly important in performance-critical applications where resource constraint is a factor.

Using Pointers

Pointers are an essential concept in C programming, allowing developers to work with memory addresses and directly manipulate data stored in memory. Understanding how to use pointers is fundamental for tasks such as dynamic memory allocation and efficient data handling.

To declare a pointer, you specify the data type followed by an asterisk (*) and the pointer variable name. Proper syntax is crucial for the declaration to ensure the pointer variable holds the correct type of address in memory.

Once a pointer is declared, you must initialize it before use. Initialization involves assigning a valid memory address to the pointer, typically the address of a variable obtained using the address-of (&) operator.

De-referencing a pointer, also known as indirection, involves accessing the value stored in the memory location the pointer points to. This is accomplished using the asterisk (*) operator, which returns the value located at the pointer's address.

int a = 10;
// Declare and initialize integer variable
int *p = &a;
// Declare pointer p and initialize with address of a
printf("Value of a: %d, Value via pointer p: %d", a, *p);
// De-reference p to get value of a		 This example illustrates how to declare an integer variable and a pointer to it. The pointer is initialized with the address of the variable using the & operator. De-referencing using *p retrieves the variable's value.
char b = 'X';
// Declare and initialize char variable
char *q;
q = &b;
// Initialize pointer q with address of b
printf("Character: %c, Through pointer q: %c", b, *q);
// De-reference q to get value of b		 Here, a char variable is used with a pointer. The pointer is declared and later assigned the address of the char variable. De-referencing allows access to the character value.

The size of a pointer is generally constant for a given architecture, usually 4 bytes on a 32-bit system and 8 bytes on a 64-bit system, regardless of the data type it points to. This is because a pointer holds a memory address, and the size depends on the address space of the architecture.

Pass by address

In C programming, understanding how data is passed to functions is fundamental. The concept of Pass by Value is the default in C, where a copy of the actual argument is made in memory. Hence, modifications to parameter variables within a function don't reflect on the originals outside the function.

Here's a typical example of Pass by Value:

CodeExplanation
void addTen(int num) {
    // Add 10 to the local copy of num
    num += 10;
}

int main() {
    int n = 5;
    addTen(n);
    printf("%d", n);  // Prints 5
}
The function addTen creates a local copy of n. Any changes made do not affect the original n in main.

The main limitation of Pass by Value is the inability to modify original data directly from a function. Pass by Address, using pointers, is a technique that allows function modifications to reflect on the original variables.

Let's look at a swapping example using Pass by Address:

CodeExplanation
void swap(int* a, int* b) {
    int temp = *a;  // Store the value at a
    *a = *b;       // Copy value from b to a
    *b = temp;     // Copy original value of a to b
}

int main() {
    int x = 10, y = 20;
    swap(&x, &y);
    printf("%d %d", x, y);  // Prints 20 10
}
The function swap changes the values of x and y in main directly by accessing their addresses.

Another example showcases obtaining a new value via a function:

CodeExplanation
void increment(int* n) {
    *n += 1;  // Increment the value at the given address
}

int main() {
    int number = 3;
    increment(&number);
    printf("%d", number);  // Prints 4
}
This program demonstrates direct modification of number through its address by incrementing its value.
Pointer Arithmatics
A pointer in c is an address, which is a numeric value. Therefore, you can perform arithmetic operations on a pointer just as you can on a numeric value.

The only valid arithmetic operations applicable on pointers are:
  1. Increment and decrement ++ and --
  2. Addition of integer to a pointer
  3. Subtraction of integer to a pointer
  4. Subtracting two pointers of the same type
Following are Invalid pointer operations
  1. Addition of two Pinters
  2. Multiplication of two Pinters
  3. Division of two Pinters
  4. Mod of two Pinters

Function Pointer
​​​​​​​In C, like normal data pointers (int *, char *, etc), we can have pointers to functions. Following is a simple example that shows declaration and function call using function pointer.

Example:
#include <stdio.h>
// A normal function with an int parameter
// and void return type
void fun(int a)
{
    printf("Value of a is %d ", a);
}
 
int main()
{
    // fun_ptr is a pointer to function fun()  
    void (*fun_ptr)(int) = &fun;
 
    /* The above line is equivalent of following two
       void (*fun_ptr)(int);
       fun_ptr = &fun;  
    */
 
    // Invoking fun() using fun_ptr
    (*fun_ptr)(10);
 
    return 0;
}
Output:
Value of a is 10

we need an extra bracket around function pointers like fun_ptr because If we remove bracket, then the expression 'void (*fun_ptr)(int)' becomes 'void *fun_ptr(int)' which is declaration of a function that returns void pointer.

Following are some interesting facts about function pointers.
  1.  Unlike normal pointers, a function pointer points to code, not data. Typically a function pointer stores the start of executable code.
  2.  Unlike normal pointers, we do not allocate de-allocate memory using function pointers.
  3. A function's name can also be used to get functions' address. For example, in the below program, we have removed address operator '&' in assignment. We have also changed function call by removing *, the program still works.
  4.  Like normal pointers, we can have an array of function pointers.
  5.  Like normal data pointers, a function pointer can be passed as an argument and can also be returned from a function.

Pointer to Pointer
​​​​​​​In C, we can also define a pointer to store the address of another pointer. Such pointer is known as a double pointer (pointer to pointer). The first pointer is used to store the address of a variable whereas the second pointer is used to store the address of the first pointer.

syntax :
int **p; // pointer to a pointer which is pointing to an integer.
Example:
#include<stdio.h>  
void main ()  
{  
    int a = 10;  
    int *p;  
    int **pp;   
    p = &a; // pointer p is pointing to the address of a  
    pp = &p; // pointer pp is a double pointer pointing to the address of pointer p  
    printf("address of a: %u ",p); // Address of a will be printed   
    printf("address of p: %u ",pp); // Address of p will be printed  
    printf("value stored at p: %d ",*p); // value stoted at the address contained by p i.e. 10 will be printed  
    printf("value stored at pp: %d ",**pp); // value stored at the address contained by the pointer stoyred at pp  
}  

Output:
address of a: 4528734
address of p: 4528738
value stored at p: 10
value stored at pp: 10
