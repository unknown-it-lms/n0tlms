Notes: C- Conditional Statements

Conditional Statements
Many times in programming, we want to perform some tasks only if some condition is satisfied.
For example:
  • In a division operation, we perform the division only if the denominator is not zero.​​​​​​​​​​​​​​
  • In a web application, we allow the user to go to home page only if the username and password are correct.

Note that in the first case, (performing the division operation) is the task and (denominator being non-zero) is the condition for doing that task. Similarly, in the second case, (taking the user to home page) is the task and (username and password being correct) is the condition for that task.

Conditional statements in C allow you to perform such tasks by executing certain blocks of code only if some condition is true. They enable decision-making in programs, making it possible to perform different actions depending on some conditions.
if condition
Condition in C language is checked using 'if-statement'
SyntaxExplanation
if(condition)
{
   statement 1;
   statement 2;
   :
} 		Here all the statements inside the curly brackets (called as if-code-block) will be executed only if the condition is true, otherwise all the statements will be ignored.
Example:
CodeExplanation
int a=6;
if(a>0)
 {
    printf("Value of a is %d", a);
    printf("This is a positive number");
 }		The given code checks whether a is a position number or not. It checks if a is greater then 0 (This is the condition). If this is true, it executes all the statements inside the if code block. Else the entire code block is ignored. 
Note that there is not semi-colon after if (a>0)
int a=6;
if(a>0)
 printf("%d is a positive number", a);
 
Note that: 
If the else code block has only one statement, we need not write the curly brackets.
But curly brackets are necessary, if the if-code-block has more than one statements which we want to execute together, if the condition is true.

if..else
​​​​Some times, we want that some statements should be executed if a condition is true and some other statements should be executed if the condition is false. In C programming, we can handle this logical requirement using if-else condition, as follows:
SyntaxExplanation
if (condition)
{
   statement 1;
   statement 2;
   :
}
else
{
  statement 3;
  statement 4;
}		Here:
  • statement 1 and 2 will be executed if the condition is true.
  • If condition is false else part will be executed, i.e statements 3 and 4 .
  •  At any point of time either statement 1 and 2 will be executed or statements 3 and 4 will be executed.
  • else part is not compulsory. There can be if without else but there can not be else without if statements.
Example....

int a =5;
if (a>=0)
{
    printf("Number is positive");
}
else
{
    printf("Number is negative");
}		In this code, if the condition is true, "Number is positive" gets displayed on console. Else "Number is negative" gets displayed on the console.

    printf("Number is negative");		In this case, the if-codeblock as well as the else-codeblock have only one statement. Hence, we can write the same code without brackets also. This is shown below:
if..else if Ladder
There are some scenarios where we need to check multiple conditions one after another. For example, to check if a number is greater than zero, it is called as a positive number. Else if it is less than zero, it is a negative number, else it is 0 which is a neutral number (Note that zero is neither positive, nor negative. It is a neutral number). Such scenarios can be handled using if-else-if ladder. 

SyntaxExplanation
if(condition 1)
{
    statement-block1;
}
else if(condition 2)
{
    statement-block2;
}
else if(condition 3)
{
    statement-block3;
}
else
{
    default-block;
}		Here:
  • If condition 1 is true, statement-block1 gets executed (and all other blocks are ignored). 
  • If condition 2 is true, only statement-block2 gets executed (and all other blocks are ignored). 
  • If condition 3 is true, only statement-block3 gets executed (and all other blocks are ignored).
  • If all conditions are false, only the default-block gets executed. 

The expression is tested from the top (of the ladder) to downwards. As soon as the true condition will found, the statement associated with it is executed.

Note that: If any block has only one statement, curly brackets are not needed. 
Lets use this feature to test if a number entered by the user is positive or negative or zero.
CodeExplanation
#include<stdio.h>
#include<conio.h>
void main()
{
    int a;
    printf("Enter a Number: ");
    scanf("%d",&a);
    if(a > 0)
    {
        printf("Given Number is Positive");
    }
    else if(a < 0)
    {
        printf("Given Number is Negative");
    }
    else if(a == 0)
    {
        printf("Given Number is Zero");
    }
   getch();
}  		This code first checks if the given number is positive. If not, it checks if the number is negative. If not it checks if it is zero and displays the message accordingly.

Code improvement tips:
  • All code blocks have only a single statement, hence all brackets can be removed.
  • If a number of not greater than 0 and not less than 0, it is definitely equal to zero. Hence, checking the last condition is unnecessary. 
The same logic can be written as follows:

if(a > 0)
    printf("Given Number is Positive");
else if(a < 0)
    printf("Given Number is Negative");
else
    printf("Given Number is Zero");
Nested if
In some cases, inside the code block of if-else, we may have to check for some other condition using another if-else statement. Checking condition using 'if-statement' inside another 'if-statement' is called as nested if.​​​​​​
SyntaxExplanation
if (condition 1) // Outer if
 {
   if (condition 2) // Inner if
    {
      statement 1;
      statement 2;
    }
    else
    {
      statement 3;
      statement 4
    }
 }		Here, the code block of condition 1 (which is called as outer if) has another if-else statement based on condition 2 (which is called as inner if-else).
All rules of outer if-else are applicable to inner if-else also. 
Example:
Lets write a code to check if a number is positive and even.

printf("Enter a number: ");
scanf("%d", &a);
if (a>0)
{
    if(a%2==0)
    {
        printf("This is a positive even number");
    }
    else
    {
        printf("This is a positive odd number");
    }
​​​​​​​}
else if(a<0)
{
    if(a%2==0)
    {
        printf("This is a negative even number");
    }
    else
    {
        printf("This is a negative odd number");
    }
}
else
{
    printf("The number is zero");
}		Here, Using the condition a>0, we check if the number if positive or not. We further check if the number if even or odd using the condition a%2 using nested if else statements.

        printf("This is a negative even number");
    else

else
    printf("The number is zero");

Note that the entire inner if-else is treated as a single statement, hence brackets are not needed for outer if-else
Ternary operator
Ternary operator provides a short-cut notation for if-else condition. 
syntax:
SyntaxExplanation
Expression 1? Expression 2 : Expression 3If the expression 1 is true, expression 2 is executed and its value is returned. Else expression 3 is executed and its value is returned.
Using ternary operator
CodeExplanation
int max, a=5, b=2;
max = (a>b)? a: b;		Here, if a>b, value of a is returned (and assigned to max) else value of b is returned (and assigned to max)
int c, a=5, b=2;
c = (a==b)? (10+a): (20+b);		Here,
if a is equal to b, the expression (10+a) is executed and its result is returned (which gets assigned to c). Else, the expression (20+b) is executed and its result is returned (which gets assigned to c).
For the given values of a and b, value of c will be 22
(a>b) ?
    printf("a is max"):
    printf("b is max");		We can even write all the expressions on separate lines as shown here. 
In this example, if a is greater than b, the message "a is max" gets displayed else the message "b is max" gets displayed.
Nested ternary operations:
CodeExplanation
a==0 ?
    printf("Number is zero") :
    (a>0?
        printf("Number is positive") :
        printf("Number is negative")
    );		Note that the else part of outer ternary operator has inner ternary operator again. 
Comparison with if-else
The if-else statements are more readable to understand the logic but ternary operators gets executed faster than the if-else statements. So when code readability and de-bugging is needed, go for if-else and when speed of execution of program is more important, use ternary operators.
Switch Case
The if-else-if ladder can be very unreadable if we have to check for multiple conditions. To handle such scenario, C provides an elegant and clear option in the form of a switch-case statement.
SyntaxExplanation
switch( expression )
{
    case value-1:
            Block-1;
            break;
    case value-2:
            Block-2;
            break;
    case value-n:
            Block-n;
            break;
    default:
            Block-1;
            break;
}		Here
  • If the value of expression==value-1, Block-1 gets executed and all other blocks are ignored. 
  • Else if the value of expression==value-2, Block-2 gets executed and all other blocks are ignored. 
  • Else if the value of expression==value-n, Block-n gets executed and all other blocks are ignored. 
  • Else default-block gets executed.

Note that 
  • A block gets executed till it reached to break statement
  • The condition checking is done from top to bottom. Hence, we should write the default statement at the end only. Else it makes all cases written below it unreachable (which means that they will never get executed in any case)

Example:
Many times on apps similar to Mobile banking, we hear a message asking us to press a key for next action. This situation can be handled using switch-case as shown below.

CodeExplanation
#include <stdio.h>
int main()
{
    int a;
    printf("Enter 1 to continue in English,
                    2 to continue in Hindi,
                    3 to read this message again,
                    4 to quit:");

    scanf("%d", &a);
   switch (a)
   {
       case 1:
               printf("You have selected to continue in English");
               break;
       case 2:
               printf("You have selected to continue in Hindi");
                break;
       case 3:
               printf("You have selected to read this message again");
               break;
       case 4:
               printf("You have selected to quit. Good bye!");
               break;
       default: printf("Please enter a valid value");
                break;   
   }
   return 0;
} 		Here, based on the user input, the matching case is selected and its code block gets executed.
If the user input is not matching with any of the cases, the default case gets executed. 
Rules for switch:
  1. The expression provided in the switch should result in a constant value otherwise it would not be valid.
  2. We can use only short, int, char type in switch case. Float values are not allowed.
  3. Duplicate case values are not allowed.
  4. The default statement is optional. Even if the switch case statement do not have a default statement, it would run without any problem.
  5. The break statement is used inside the switch to terminate a statement sequence. When a break statement is reached, the switch terminates, and the flow of control jumps to the next line following the switch statement.
  6. The break statement is optional. If omitted, execution will continue on into the next case. The flow of control will fall through to subsequent cases until a break is reached.
  7. Nesting of switch statements are allowed, which means you can have switch statements inside another switch. However nested switch statements should be avoided as it makes program more complex and less readable.
