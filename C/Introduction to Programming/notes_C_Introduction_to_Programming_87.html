Basics of programming languages
A program is a set of step-by-step instructions given to a computer to perform certain task. It is like we are communicating with the computer, requesting it to perform the tasks we want. For every communication we requite a language. The language we use for communicating with the computer, by writing our programs, is called as a programming language.
Classification of programming languages
Programming languages are classified into various catagories on the basis of various criterions as follows:
  1. Classification by Abstraction Level
  2. Classification by Execution Method
  3. Classification by Programming Paradigm

1. Classification by Abstraction Level
Abstraction level of a programming language is the level of its closeness to human language. If a programming language is close to human language, it is considered as more abstract language. But if a programming language is less close to human language but closer to machine language, it is considered as less abstract programming language. On the basis of this criterio, programming languages are classified as follows:
TypesDetails
Low-Level Languages (Closer to Hardware):Machine Language (1st Gen):  This is the language that the CPU understands directly. Its written in binary code. 

Assembly Language (2nd Gen): This language uses some symbols to represent machine instructions
High-Level Languages (Closer to Human Language)Use English-like keywords and syntax, making them easier to read, write, and maintain.
Third Generation Language (3rd Gen): High-Level Procedural Languages. (C, C++, Java, Fortran)

Fourth Generation Language (4th Gen): Very High-Level / Non-Procedural Languages. (SQL, MATLAB, Visual basic)

Fifth Generation Language (5th Gen): Constraint-Based Languages (Pytho, Prolog, Lisp, Mercury)

2. Classification by Execution Method
The program written in every language must be translated to binary language, the languge computer understands. This process of convering programming language code into machine code is called as execution method. Based on this, programming languages are classified into following catagories.
Type of execution methodProcessExamples
Compiled languagesEntire source code is translated into an executable machine code file before execution. This is done by the compiler. The execution is faster but it needs recompilation after every change in code.C, C++
Interpreted languagesThe source code is translated and executed line-by-line at runtime. This is done by the interpreter. This is slower in execution but eliminated the need of compilation after every change in source code.Python, JavaScript, PHP, Ruby
Hybrid (JIT)Compiles source code to some intermediate language first, which is then translated Just-In-Time (JIT), to machine code during execution.Java, C#


3. Classification by Programming Paradigm
Programming paradigm is basically the way a software program is designed. Various programming languages are classified on the basis of this criterio as follows:
TypesStyle of program designExample
Procedural languagesA program is simply sequence of instructions grouped in functions. C, Fortran, Pascal
Object oriented programming languages (OOP)The program is written in the form of objects and classes. Such program design is more closer to real-life scenarion.C++, Java, C#, Python
History of programming languages
Programming languages have evolved from very low-level to very high-level over a long period of time as shown below:
PeriodTypes Examples
Early Beginnings (1840s - 1940s)It stared during this period with programming languages that are very close to machine.Assembly languages
High-Level Languages (1950s - 1960s)This era focused on languages closer to human language, making programming faster and less error-prone.FORTRAN, LISP, COBOL, BASIC
Structured Programming Languages (1970s)More abstract in nature. More use of English-like keywordsC, SQL, Pascal
Object-Oriented Revolution (1980s - 1990s)Programming style changed from sequence of instructions to making the software look more like real-life objects.C++, Java, Python, Javascript
Modern and Specialized Languages (2000s - Present)Include features that handle more modern requirements like Web, Mobile etcC#, GO, Swift, Rust

Memory and Storage Units
Computer stores every piece of information strictly in the form of either 0 or 1. The basic unit of memory used to hold this 0 or 1 is called as 1 bit of memory.
  • Bit                           Basic unit of memory. Stored either 0 or 1
  • Byte                        1 byte is 8 bits
  • Kilobyte (KB)         1 KB is 1024 Bytes
  • Megabyte (MB)    1 MB is 1024 KB
  • Gigabyte (GB)       1 GB is 1024 MB
  • Terabyte (TB)        1 TB is 1024 GB

Basics of programming
Every program is a set of instructions given to the computer in a logical manner to achieve some task. The most important part of programming is to decide this logical flow in instructions. This logical flow i sindependant of any programming language. Once this is done, it can be converted into a program using any suitable programming language.
This logical flow of instrcitions is called as algorithm and its visual representation is called as a flow-chart. 
Algorithm
  • An algorithm is a finite sequence of well-defined, step-by-step instructions for solving a specific problem or accomplishing a computational task.
  • It is the core logic for a program.
  • It is written in a natural or semi-formal language (like pseudocode), independent of any specific programming language.
  • Note that for writting an algorithm, you need not have knowledge of any programming language.
Flowchart
  • A flowchart is the graphical or pictorial representation of an algorithm with the help of different symbols, shapes and arrows in order to demonstrate a process or a program.
  • With algorithms, we can easily understand a program.
  • The main purpose of a flowchart is to analyze different processes. Several standard graphics are applied in a flowchart.
