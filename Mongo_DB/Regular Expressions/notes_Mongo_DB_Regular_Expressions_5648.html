Notes: Mongo_DB- Regular Expressions

Regular Expressions
Regular Expressions are used in all languages to search a pattern or word in any string.  MongoDB uses Perl-compatible regular expressions(PCRE) version 8.42 along with UTF-8 support.
A regular expression is a generalized way to match patterns with sequences of characters.
Concept
The real power of regular expressions doesn't come from simple word searches, but rather by using special regular expression syntax. For example, a single period (.) in a regular expression represents any single character.

  • With $regex Operator
  • Without $regex Operator
​​​In MongoDB we can do pattern matching in two different ways 
  
The $regex operator in MongoDB is a powerful tool that provides regular expression capabilities for pattern matching within strings in queries. It is particularly useful when the exact field value is unknown which allows for flexible and efficient searches within collections.
 
Notations used in regular expression..
NotationDescription
^beginning of string
$end of string
\ddigit. Set of number(numeric value).Same as[0-9]
[0-9]set of number(numeric value). Same as \d
[Aa]set of alphabet.
/ /pattern 
   
Following is the syntax for regular expression.

{ <field>: { $regex: /pattern/, $options: '<options>' } }
{ <field>: { $regex: 'pattern', $options: '<options>' } }
{ <field>: { $regex: /pattern/<options> } }
Using Regular Expressions  
  • []
    --Display all documents where actor name is amitabh or Amitabh.
    >db.movie.find({actor:/^[Aa]mitabh/})

  • [0-9]
    --Display all documents where movie name starts with digit.
    >db.movie.find({actor:/^[0-9]/})

  • $
    --Display all documents where movie name ends with digit.
    >db.movie.find({actor:/[0-9]$/})

  • \d
    --Display all documents where movie name starts with digit.
    >db.movie.find({actor:/^\d.*$/})
    --Display all documents where movie name starts with digit and ends with N.
    >db.movie.find({actor:/^\d.*N$/})

  • $eq match on a regular expression
    An explicit query using $eq and a regular expression will only match an object which is also a regular expresssion.  
    >db.collection.find( { company: { $eq: /MongoDB/ } }, {_id: 0 } )

  • $regex
    using  $regex following query can be implamented as follows.
    db.collection.find( { company: /MongoDB/ }, {_id: 0 })
    db.collection.find( { company: { $regex: /MongoDB/ } }, {_id: 0 }

    Following query selects all documents in the inventory collection where the item field value does not start with the letter 'p'.
    db.inventory.find( { item: { $not: { $regex: /^p.*/ } } } )

  • Using regex Expression with Case Insensitive
          To make the search case insensitive, we use the $options parameter with value $i. The following command             will look for strings having the word tutorialspoint, irrespective of smaller or capital case −
           
         >db.posts.find({post_text:{$regex:"Know-it",$options:"$i"}})

          Here pattern "Know-it will matched and it will not be case sensitive match.
