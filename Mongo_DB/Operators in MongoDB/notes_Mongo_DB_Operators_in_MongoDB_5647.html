Concept 
 MongoDB has many operators . It is the operators which has made MongoDB a powerful data storage. Writing complex queries becomes easy with the operators. Following are the types of operators......
  • Query and Projection Operators
    Query operators provide ways to locate data within the database and projection operators modify how data is  presented.
  • Update Operators
    Update operators are operators that enable you to modify the data in your database or add additional data.
  • Aggregation Pipeline Stages
    Available aggregation stages for Aggregation Pipeline.
  • Aggregation Pipeline Operators
    Aggregation pipeline operations have a collection of operators available to define and manipulate documents in pipeline stages.   
  • Comparison operators

Using Operators
Single or multiple conditions can be given while writting the queries.
e.g
Simple Condition 
--Display all the movies in film collection having rating=6
>db.movie.find({rating:6})

Multiple conditions 
--Dispaly all movies with rating 3 and price 200
>db.movie.find({rating:3,price:200})
Here, comma(,) works like logical 'and' where both the conditions are true.
Query and Projection Operators
MongoDB provides query and projection operators. These operators play a crucial role in filtering and shaping data returned from MongoDB collections.  

Query Operators
Query operators are used to filter the documents in a MongoDB collection based on specific conditions. They are applied to the find() method to return only the documents that match the specified criteria. Here are some common query operators in MongoDB
OperatorMeaningExample
$gtgreater than operator--Display all movies with rating > 3
>db.movie.find({rating:{$gt:3},{price:200}})
$lt and $gt multiple conditions using $gt and $lt--Display all movies having price > 200 and <400
>db.movie.find({price:{$gt:200,$lt:400}})
$gtequals operator 
Match on numeric value
--Display all documents where qty = 20.
>db.inventory.find( { qty: { $eq: 20 } } )
Match on a string value
 >db.collection.find( { company: { $eq: "MongoDB" } }, {_id: 0 } ) 
same as, 
>db.collection.find( { company: "MongoDB" }, {_id: 0 })
    returns the following result:
    { "company" : "MongoDB" }
$inset of values --Display all movies with price =200 or price =300 or price=40
>db.movie.find({price:{$in:[200,300,400]})
$nin                  

not in the set-- Display all movies  whose price is note  equal to 400 or 300 or 200 (not in operator)
 >db.movie.find({price:{$nin:[400,300,200]}})
$or 

or operator--Display all movies with rating 6 or price >400
>db.movie.find({$or:[{rating : 6},{price : {$gt:400}}])
Explanation 
Conditions  rating:6  price:{$gt:400}   
$or:[{},{},{},{}]
$not  

not operator--Display  all movies not in range 200 and 400
>db.movies.find({price:{$gt:200,$lt:400}})
Converting above query to not.
>db.movies.find({price:{$not:{$gt:200,$lt:400}}})
 $mod  

mod Operator  --Display all movies where rating is even
>db.movies.find({rating:{$mod:[2,0]}})
--Display all movies where rating is odd
>db.movies.find({rating:{$mod:[2,1]}})       
$type  


type Operator
$type operator is used to check datatype.		--Display all the movies where rating given is in character format.
>db.movie.find({rating:{type:'string'}})
--Display all the movies where rating given is not in character format.
>db.movie.find({rating:{$not : {type:'string'}}})
$exist  exist Operator
MongoDB is unstructured database.  Document may contain additional field or may not contain some specific field. We can check existing field by above operator.		--Display all the documents where rating key exists.
>dn.movie.find({rating:{$exist:true}})
--Display all the documents where rating key dose not exist.
>dn.movie.find({rating:{$exist:false}})

Handling nulls  
MongoDB handles null values very efficiently.
--Display all movies with rating null
>db.movie.find({rating:null})

Logical Operators  
  • $or (or operator)  
    --Display all movies with rating 6 or price >400
    >db.movie.find({$or:[{rating : 6},{price : {$gt:400}}])
    Explaination::
    Conditions  rating:6  price:{$gt:400}   
    $or:[{},{},{},{}]
  • $and (and operator) 
    AND Queries With Multiple Expressions Specifying the Same Field 
     db.inventory.find( { $and: [ { price: { $ne: 1.99 } }, { price: { $exists: true } } ] }
     Now consider the below  query
     db.inventory.find( {$and: [ { $or: [ { qty: { $lt : 10 } }, { qty : { $gt: 50 } } ] },
                                                     { $or: [ { sale: true }, { price : { $lt : 5 } } ] }
                                                    ]} )
  • $not (not operator) 
    db.inventory.find( { price: { $not: { $gt: 1.99 } } } ) 
  • $nor (nor operator) 
    Performs a logical NOR operation on an array of one or more query expression and selects the documents that fail all the query expressions in the array.
    db.inventory.find( { $nor: [ { price: 1.99 }, { sale: true } ]  } )
    This query will return all documents that
           1.  contain the price field whose value is not equal to 1.99 and contain the sale field whose value is not equal to true or
           2.  contain the price field whose value is not equal to 1.99 but do not contain the sale field or
           3.  do not contain the price field but contain the sale field whose value is not equal to true or
           4.  do not contain the price field and do not contain the sale field
