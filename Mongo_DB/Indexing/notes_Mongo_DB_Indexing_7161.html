Notes: Mongo_DB- Indexing

Indexing

An index is a special lookup table that the database search engine can use to speed up data retrieval. It's like an index in a book, allowing the database to quickly find documents based on specific  field values without having to scan the entire collection.    


 
Concept of index   
Indexes are special data structures that store a small portion of the collection's data set in an easy-to-traverse form. MongoDB indexes use a B-tree data structure.
Indexes support efficient execution of queries in MongoDB. Without indexes, MongoDB must scan every document in a collection to return query results. If an appropriate index exists for a query, MongoDB uses the index to limit the number of documents it must scan.      

Although indexes improve query performance, adding an index has negative performance impact for write operations. For collections with a high write-to-read ratio, indexes are expensive because each insert must also update any indexes. Also, certain restrictions apply to indexes, such as the length of the index keys or the number of indexes per collection etc. 
If your application is repeatedly running queries on the same fields, you can create an index on those fields to improve performance. For example, consider the following scenarios:
   
NoScenarioIndex Type
1A human resources department often needs to look up employees by employee ID. You can create an index on the employee ID field to improve query performance.Single Field Index
2A salesperson often needs to look up client information by location. Location is stored in an embedded object with fields like state, city, and zipcode. You can create an index on the location object to improve performance for queries on that object.Single Field Index on an embedded document
When you create an index on an embedded document, only queries that specify the entire embedded document use the index. Queries on a specific field within the document do not use the index. 
3A grocery store manager often needs to look up inventory items by name and quantity to determine which items are low stock. You can create a single index on both the item and quantity fields to improve query performance.Compound Index
Default Index
MongoDB creates a unique index on the _id field during the creation of a collection. The _id index prevents clients from inserting two documents with the same value for the _id field. You cannot drop this index.


Creating index
You can create and manage indexes in MongoDB Atlas, with a driver method, or with the MongoDB Shell. MongoDB Atlas is the fully managed service for MongoDB deployments in the cloud.

Index Names
The default name for an index is the concatenation of the indexed keys and each key's direction in the index (1 or -1) using underscores as a separator. For example, an index created on { item : 1, quantity: -1 } has the name item_1_quantity_-1.
Create and Manage Indexes with a MongoDB Shell
Create an Index
To create an index, use the createIndex() shell method.  
This methodcreates the index if an index of the same specification does not exist.
syntax
>db.collection.createIndex( <key and index type specification>, <options> )
>db.collection.createIndex().
Example
This example creates a single key ascending index on the name field 
db.collection.createIndex( { name: 1 } )

Check existing indexes 
To see what indexes exist on the collection 
>db.collection.getIndexes()

Types of Indexes
Different index types support different types of data and queries in MongoDB.

1) Single Field Index
Single field indexes collect and sort data from a single field in each document in a collection.

Single field indexes store information from a single field in a collection. By default, all collections have an index on the _id field. Additional indexes can be used to speed up important queries and operations.

Single-field index can be created on any field in a document, including
  • Top-level document fields
  • Embedded documents
  • Fields within embedded documents

When a index is created, following things are specified
  • The field on which to create the index.
  • The sort order for the indexed values (ascending or descending).
  • A sort order of 1 sorts values in ascending order.
  • A sort order of -1 sorts values in descending order.

Syntax
db.<collection>.createIndex( { <field>: <sort-order> } )

2)  Compound Index
Compound indexes collect and sort data from two or more fields in each document in a collection. Data is grouped by the first field in the index and then by each subsequent field.

Syntax

db.<collection>.createIndex( {
                              <field1>: <sortOrder>,
                              <field2>: <sortOrder>,
                              ...
                              <fieldN>: <sortOrder>
                              } )
Example
Consider the following example
db.students.insertMany([
   {
      "name": "Alice",
      "gpa": 3.6,
      "location": { city: "Sacramento", state: "California" }
   },
   {
      "name": "Bob",
      "gpa": 3.2,
      "location": { city: "Albany", state: "New York" }
   }
])

Creating compound index containing the name and gpa fields 

db.students.createIndex( {
   name: 1,
   gpa: -1
} )

In this example 
The index on name is ascending (1).
The index on gpa is descending (-1).

Results
The created index supports queries that select on:
Both name and gpa fields.
Only the name field, because name is a prefix of the compound index.

For example, the index supports these queries:

db.students.find( { name: "Alice", gpa: 3.6 } )
db.students.find( { name: "Bob" } )
