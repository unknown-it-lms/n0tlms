Notes: Mongo_DB- CRUD Operations

CRUD Operations 
The basic methods of interacting with a MongoDB server are called CRUD operations. CRUD stands for Create, Read, Update, and Delete. These CRUD methods are the primary ways you will manage the data in your databases.
  
CRUD operations describe the conventions of a user-interface that let users view, search, and modify parts of the database.

MongoDB documents are modified by connecting to a server, querying the proper documents, and then changing the setting properties before sending the data back to the database to be updated. CRUD is data-oriented, and it’s standardized according to HTTP action verbs.
Implementing CRUD operations  
  • Create operation is used to insert new documents in the MongoDB database.
  • Read operation is used to query a document in the database.
  • Update operation is used to modify existing documents in the database.
  • Delete operation is used to remove documents in the database.

Performing  CRUD Operations  
Let us see how to carry out the individual operations and manipulate documents in a MongoDB database. 
  • Create Operations ::
    For MongoDB CRUD, if the specified collection doesn’t exist, the create operation will create the collection when it executes. Create operations in MongoDB targets a single collection, not multiple collections. Insert operations in MongoDB are atomic on a single document level.

    MongoDB provides two different create operations that you can use to insert documents into a collection

    1. db.collection.insertOne()
         insertOne() allows user to insert one document into the collection. 
    2. db.collection.insertMany()
         it is possible to  insert multiple items at one time by calling the insertMany() method on the desired collection. In this case, we pass multiple items into our chosen collection and separate them by commas. Within the parentheses, we use brackets to indicate that we are passing in a list of multiple entries. This is commonly referred to as a nested method.

  • Read Operations :: 
    The read operations allow you to supply special query filters and criteria that let you specify which documents you want. The MongoDB documentation contains more information on the available query filters. Query modifiers may also be used to change how many results are returned.

    MongoDB has two methods of reading documents from a collection:

    1. db.collection.find()
         In order to get all the documents from a collection, we can simply use the find() method on our chosen    collection. Executing just the find() method with no arguments will return all records currently in the                collection.
    2. db.collection.findOne()
         In order to get one document that satisfies the search criteria, we can simply use the findOne() method on our chosen collection. If multiple documents satisfy the query, this method returns the first document according to the natural order which reflects the order of documents on the disk. If no  documents satisfy the search criteria, the function returns null.  

  • Update Operations ::
    Same like create operations, update operations operate on a single collection, and they are atomic at a single document level. An update operation takes filters and criteria to select the documents that user wants to update.
    One should be careful when updating documents, as updates are permanent and can’t be rolled back. This applies to delete operations as well.

     MongoDB has three methods of updating documents:

    1. db.collection.updateOne()
       ​​​​​​​ Here, user can update a currently existing record and change a single document with an update                      operation. To do this, updateOne() method is used on a chosen collection . To update a document, we            provide the method with two arguments, an update filter and an update action. The update filter defines which items we want to update, and the update action defines how to update those items.  
   2. db.collection.updateMany()
       updateMany() allows us to update multiple items by passing in a list of items, just as we did when                   inserting multiple items. This update operation uses the same syntax for updating a single document.
   3. db.collection.replaceOne()
       replaceOne() method is used to replace a single document in the specified collection. replaceOne()                replaces the entire document, meaning fields in the old document not contained in the new will be lost.
​​​​​​​
  • Delete Operations ::
    Delete operations operate on a single collection, like update and create operations. Delete operations are also atomic for a single document. You can provide delete operations with filters and criteria in order to specify which documents you would like to delete from a collection. The filter options rely on the same syntax that read operations utilize.

    MongoDB has two different methods of deleting records from a collection:

    1. db.collection.deleteOne()
         deleteOne() is used to remove a document from a specified collection on the MongoDB server. A filter           criteria is used to specify the item to delete. It deletes the first record that matches the provided filter.
    2. db.collection.deleteMany()   
         is a method used to delete multiple documents from a desired collection with a single delete operation.       A list is passed into the method and the individual items are defined with filter criteria as in deleteOne().

Sorting the data 
 We can sort the data in ascending or descending order in MongoDB. Method sort() is used to perform the task.
--Display all the movies of Amitabh in ascending order of price. Here we need to sort the data on price.
> db.movie.find({actor:'Amitabh'},{name:1,price:1,_id:0}).sort({price:1})

--Display all the movies of Amitabh in descending order of price.
> db.movie.find({actor:'Amitabh'},{name:1,price:1,_id:0}).sort({price:-1})
Restricting the document display 
It is possible to put a restriction no the number of documents to be displayed. Method limit() is used to display required number of documents.
--Display only 1 movie of Amitabh.
> db.movie.find({actor:'Amitabh'},{name:1,price:1,_id:0}).limit(1)
Method skip()

Update     
One of the important operations on database is update operation. Here user can make changes in the existing data.
There are multiple operators given by MongoDB which are use to perform update operations.
Update in MongoDb is used to modify an existing document/documents in the collection.  To update the document, MongoDb has a update() method. This method can modify specific fields of an existing document/ documents or replace an existing document entirely, depending on the update parameter.
Method used to update collection is as below...
 db.collection.update()
By default, this updates a single document. By including the option multi: true , multiple documents that match the query criteria will be updated.   

update method can be used in following 3 ways
  • update()  - only matching records.
  • updateOne()  - only one matching document.
  • updateMany() - all matching documents.
Let us see how can we perform Update operations on collection.
Syntax 
1. db.collection.update({query},{update actions},{options})
2. db.collection.updateOne({query},{update actions})
3. db.collection.updateMany({query},{update actions})

Updating single document
>db.movie.update({},{$set:{ratings:5}})

Updating multiple documents  
>db.movie.update({},{$set:{ratings:5}},{multi:True})

If multi =True  :: changes are made in all documents.
              =False :: changes are made only in first document.

To update the data in the collection, Mongo uses multiple operators. Following is the list and the function of update operators.
   Operators                Meaning
$setSets the value of a field in a document.
$unsetRemoves the specified field from a document
$maxOnly updates the field if the specified value is greater than the existing field value.
$minOnly updates the field if the specified value is less than the existing field value.
$incIncrements the value of the field by the specified amount.
$mulMultiplies the value of the field by the specified amount.
$renameRenames a field.
$setOnInsertSets the value of a field if an update results in an insert of a document. Has no effect on  update operations that modify existing documents.
$unset
$currentDateSets the value of a field to current date, either as a Date or a Timestamp.

Following are considered to be the update operations 
  • Adding new key-value pair ($set).
  • Delete existing key ($unset).
  • Overwrite key-value.
  • Increasing/decreasing key-value.
                                                                                                                                                              
Using update operators 
$inc Operator(increment operator) 
  • increment the value 
           --update rating to 5 and increase price by 20 for all movies where name has 'kahani' word in it.
            >db.movie.update({name:/kahani/},{$set:{rating:5},$inc:{price:20}},{multi:true})
  • decrement the value::
           --decrease price of all movies by 100.
            >db.movie.update({name:/kahani/},{$inc:{price:-20}},{multi:true})
 
$mul Operator(multiply operator) 
--increase value by 10% of price 
>db.movie.update({name:/kahani/},{ $mul:{price:1.10}},{multi:true})

$unset (Deleting a key value pair) 
We can delete a key-value pair (refered as column in structured database).
--delete key-value pair actors for the documents  where movie name is 'Kahani'
>db.movie.update({name:/kahani/},{$unset:{actors : ""}},{multi:true})     

$currentDate() 
--increase price by 30 for all the movies with rating >4 and also , store the date on which the price is changed and also store the reason of the change.
>db.movie.update({rating:{$gt:4}},{$inc:{price:30},$currentDate:{"changePrice.date":true},$set:{"changePrice.reason":"Public Demand"}},{multi:true})

$min Operator  
--update rating to 4 if the current rating >4
>db.movie.update({name:/kahani/},{$min:{rating:4}},{multi:true}) 
Delete
In MongoDB a user can delete a collection, database or a document. This deletion is a permanent deletion and the deleted data/object cannot be retrived.

Deleting the collection    
Any existing Collection can be deleted. This deletion is permanent. Structure and data both are deleted. 
>db.Student.drop()
deletes student collection.
Deleting the Database   
>db.dropDatabase()
will delete current database.  
Deleting a document 
In MongoDB, removing data can be achieved through various methods, primarily deleteOne(), deleteMany() and remove().
deleteOne() removes a single document matching a specified filter.
deleteMany() removes all documents that match the filter.
remove() method is available but is deprecated newer versions.
To remove all documents from a collection, an empty filter can be used with deleteMany(), or the remove() method can be used with an empty filter.
 
--We can delete the document with following query
>db.student.remove({course:"BA"})
All documents with course value "BA" wil be deleted.

>db.student.deleteMany({course:"BA"})
All documents with course value "BA" wil be deleted.

>db.student.deleteOne({course:"BA"})
First  document with course value "BA" will be deleted.
