Handling Array in Mongo
Array is a collection of elements of same types. Elements get a contiguous allocation in the memory.
​​​​​​​
Concept and operators
To handle the arrays, MongoDB has multiple opeators. Followig operators are used to perform actions.
OperatorsMeaning
$ Acts as a place holder to update the first element that matches the query condition.
$[]Acts as a placeholder to update all elements in an array for the documents that match the query condition.
$[<identifier>]Acts as a placeholder to update all elements that match the arrayFilters condition for the documents that match the query condition.
$addToSetAdds elements to an array only if they do not already exist in the set.
 $popRemoves the first or last item of an array.
 $pullRemoves all array elements that match a specified query.
$pullAllRemoves all matching values from an array
$pushAdds an item to an array.
Update Operator Modifiers
ModifierMeaning
$eachModifies the $push and $addToSet operators to append multiple items for array updates.
$position Modifies the $push operator to specify the position in the array to add elements.
$sliceModifies the $push operator to limit the size of updated arrays.
$sortModifies the $push operator to reorder documents stored in an array
Using Operators
Consider the following collection 'students'.
db.students.insertMany( [
   { "_id" : 1, "grades" : [ 85, 80, 80 ] },
   { "_id" : 2, "grades" : [ 88, 90, 92 ] },
   { "_id" : 3, "grades" : [ 85, 100, 90 ] }
                                          ])
Let us see how operators work on above collection.
$ Operator 
$ acts as a place holder to update the first element that matches the query condition. It is a positional operator.
--Update the first element whose value is 80 to 82 in the grades array, using positional $ operator.
>db.students.updateOne( { _id: 1, grades: 80 }, { $set: { "grades.$" : 82 } })
 
Note that , here position of element 80 which is to be replaced, is not known. Also, the positional  $operator acts as a placeholder for the first match of the update query document.
Update Documents in an Array
The positional operator $ can also be used to update an array that contain embedded documents. To access the fields in the embedded documents, dot notation is used
Syntax
db.collection.updateOne({ <query selector> },{ <update operator>: { "array.$.field" : value } })

Consider the following document in the students collection whose grades element value is an array of embedded documents....
{
  _id: 4,
  grades: [
     { grade: 80, mean: 75, std: 8 },
     { grade: 85, mean: 90, std: 5 },
     { grade: 85, mean: 85, std: 8 }
  ]
}
Here positional $ operator is used to update the std field of the first array element that matches the grade equal to 85  .
db.students.updateOne(
   { _id: 4, "grades.grade": 85 },
   { $set: { "grades.$.std" : 6 } }
)
After the operation, the document has the following updated values....
{
   "_id" : 4,
   "grades" : [
      { "grade" : 80, "mean" : 75, "std" : 8 },
      { "grade" : 85, "mean" : 90, "std" : 6 },
      { "grade" : 85, "mean" : 85, "std" : 8 }
   ]
}
$[] Operator  
$[] acts as a placeholder to update all elements in an array for the documents that match the query condition.
--increment marks by 10 where marks as an array.
>db.student.update({},{$inc:{"marks.$[]":10}})
--incrementing with condition
>db.student.update({mark:$in:[23,32,31,20]},{$inc:{"marks.$[]":10}},{multi:true}) 
 
$push (adding new element in array) 
--Add new actor in the movie collection.
>db.movie.update({name:"Sholay"},{$push:{actor:"Hema"}})
Actor Hema will be added at the end.  $push will add new element at the end of array.
$pop (deleting value from array) 
$pop is used to delete values from array.
Deleting last value
>db.movie.update({name:"Sholay"},{$pop:{actor:1}})
Deleting first value
 >db.movie.update({name:"Sholay"},{$pop:{actor:-1}})
$pull (deleting  value)  
  • Deleting specific value
    $pull deletes specific value from array.
    >db.movie.update({name:"Sholay"},{$pull:{actor:"Sachin"}})
  • Deleting multiple values
    >db.movie.update({name:"Sholay"},{$pull:{actor:{$in:["Sachin","Hema"]}}})
$size Operator 
 $size operator is used to check the size of an array.
--Display all movies where there is entry of 4 actors.
>db.movie.find({actor:{$size:4}})    
$each Operator(adding multiple elements in array)  
--To add multiple elements in array , $each operator is used with $push.
>db.movie.update({name:"Sholay"},{$push:{actor:{$each:["A.K.Hangal","Sachin"]}}})
$position Operator(adding element at a perticular position in array)  
>db.movie.update({name:"Sholay"},{$push:{actor:{$each:["A.K.Hangal","Sachin"],$position:1}}})
Here, actors will be added at position 1. "A.K.Hangal" will be added at position 1 , "sachin" will be added at position 2 and then followed by remaining elements already present in the array.
​​​​​​​Searching data on index position 
--Display all the movies of Amitabh, where entry of Amitabh is at 1st index position.
>db.movie.find({'actor.1':'Amitabh'})
Checking by position 
--Display all restaurants with score > 9 at first position.
>db.restaurant.find({"grades.0.score":{$gt:9}})   
Matching element in array
Finding  a complete object match(document)  in the collection 
Every document in a MongoDB collection is treated as an object.
elemMatch()  is used to match entire document.
​​​​​​​
Following is an example..
db.restaurant.find({grades : {elemMatch:{grade:'A',Score:13}}})
Array filters
$filter is used to filter the documents in array.
Selects a subset of an array to return based on the specified condition. Returns an array with only those elements that match the condition. The returned elements are in the original order.

$filter
 has the following syntax:
{
   $filter:
      {
         input: <array>,
         cond: <expression>,
         as: <string>,
         limit: <number expression>
      }
}
FieldSpecification
inputAn expression that resolves to an array.
condAn expression that resolves to a boolean value used to determine if an element should be included in the output array. The expression references each element of the input array individually with the variable name specified in as.
asOptional. A name for the variable that represents each individual element of the input array. If no name is specified, the variable name defaults to this.
limitOptional. A number expression that restricts the number of matchiOptional. A number expression that restricts the number of matching array elements that $filter returns.
Note ::   limit less than 1 cannot be specified. The matching array elements are returned in the order they appear in the input array. If the specified limit is greater than the number of matching array elements  $filter returns all matching array elements. If the limit is null, $filter returns all matching array elements. 
 
Example ::
A collection sales has the following documents:

db.sales.insertMany( [
   {
      _id: 0,
      items: [
         { item_id: 43, quantity: 2, price: 10 },
         { item_id: 2, quantity: 1, price: 240 }
      ]
   },
   {
      _id: 1,
      items: [
         { item_id: 23, quantity: 3, price: 110 },
         { item_id: 103, quantity: 4, price: 5 },
         { item_id: 38, quantity: 1, price: 300 }
      ]
   },
   {
      _id: 2,
      items: [
         { item_id: 4, quantity: 1, price: 23 }
      ]
   }
] )

The following example filters the items array to only include documents that have a price greater than or equal to 100:

db.sales.aggregate( [
   {
      $project: {
         items: {
            $filter: {
               input: "$items",
               as: "item",
               cond: { $gte: [ "$$item.price", 100 ] }
            }
         }
      }
   }
] )

The operation produces the following results:

{
   "_id" : 0,
   "items" : [
      { "item_id" : 2, "quantity" : 1, "price" : 240 }
   ]
}
{
   "_id" : 1,
   "items" : [
      { "item_id" : 23, "quantity" : 3, "price" : 110 },
      { "item_id" : 38, "quantity" : 1, "price" : 300 }
   ]
}
{ "_id" : 2, "items" : [ ] }
