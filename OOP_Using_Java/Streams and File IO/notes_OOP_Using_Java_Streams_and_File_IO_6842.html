Streams and File IO
We know that a program receives input, processes it as per the business logic and then generates output. Till now, we have seen how to process the input using various programming constructs and OOP concepts.
Now, in this topic, we will understand different ways in which our program can receive input from various input sources and write the output to various output destinations.
Concept of stream
In both the process of reading data from sources as well as writing it to various destinations, it is the data which is in flow. This flowing data is called as stream.
​​​​​​​
Following data flows are examples of stream:
  • Data entered by the user through keyboard and moving to memory of a program
  • Data moving from the program's memory to output device like monitor
  • While reading data from a file, the data moving from file on hard-disk to program's memory
  • While writing data to a file, the data moving from program's memory to file on hard-disk 

Based on how we interpret this flowing data, there are two types of streams in Java:
  1. Byte stream: It treats the data as flow of 8-bit bytes. 
  2. Character stream: It treats the data as flow of 16-bit Unicode characters. They are used for text data IO.

Lets now study Byte streams and Character streams in detail.

Using Byte streams
Note that Byte stream classes treat the data as flow of 8-bit bytes. They don't interpret the data as String, float, double etc. Hence, they accept the data as Byte[] while writing and return the data as Byte[] after reading. We have to manage conversion from byte[] to Built-in data types and vice versa.

Lets now see following scenarios
  1. Writing to file using FileOutputStream
  2. Read from file using FileInputStream
  3. Use of BufferedOutputStreamwith FileOutputStream to improve writing performance
  4. Use of BufferedInputStream with FileInputStream to improve reading performance

1. Writing to file using FileOutputStream
FileOutputStream out = null;

catch(Exception ex) {

}
finally {

}		Note that:
We can't write string directly using FileOutputStream class. We have to get Byte[] from string using str.getBytes() and then write that byte[] using write() method.
Also note that we have to close the stream in any case. Hence its written in finally block.
2. Read from file using FileInputStream
FileInputStream in = null;
try {

}
finally {

}		Note that:
All read operations of FileInputStream class, return stream of data in the form of byte[]. We have to convert it to built-in data types in our program.
Also note that we have to close the stream in any case. Hence its written in finally block.

Using Character streams
Note that Character stream classes treat the data as flow of 16-bit Unicode characters. Hence, these classes are more suitable for reading and writing text data.

  2. Read text from file using FileReader

Pre-defined streams
All the programming languages provide support for standard I/O where the user's program can take input from a keyboard and then produce an output on the computer screen. Java provides the following three standard streams:
  • Standard Input − This is used to provide the data from the standard input device (keyboard), to user's program . This stream is represented as System.in.
  • Standard Output − This is used to output the data produced by the user's program to the standard output which is computer screen. This stream is represented as System.out.
  • Standard Error − This is used to output the error data produced by the user's program to computer screen. This is represented as System.err.

File class
The File class is an abstract representation of file and directory pathname. A pathname can be either absolute or relative. This class can be used for creating new file, deleting the file, checking whether the given pathname is a file or directory as well as getting mode information about the file or directory.

boolean  canExecute()It tests whether the application can execute the file denoted by this abstract pathname.
boolean  canRead()It tests whether the application can execute the file denoted by this abstract pathname.
boolean isDirectory() It tests whether the file denoted by this abstract pathname is a directory.
boolean  isFile()  It tests whether the file denoted by this abstract pathname is a normal file.

String[]  list(FilenameFilter filter) It returns an array of strings naming the files and directories in the directory denoted by this abstract pathname that satisfy the specified filter.
boolean mkdir() It creates the directory named by this abstract pathname.
longlength()    Returns the length of the file denoted by this abstract pathname.
longlastModified()Returns the time that the file denoted by this abstract pathname was last modified.

Serialization
  • Serialization is a mechanism of writing the state of an object into a byte-stream. The reverse operation of serialization is called deserialization where byte-stream is converted into an object.
  • The serialization and deserialization process is platform-independent, it means you can serialize an object on one platform and deserialize it on a different platform.
  • It is mainly used in Hibernate, RMI, JPA, EJB and JMS technologies.
  • For serializing the object, we use ObjectOutputStream class and call the writeObject() method on it.
  • For deserialization we user ObjectInputStream class and call the readObject() method on it. 
  • The class of object we want to serialize or deserialize, must implement an interface Serializable. This interface doesn't have any method inside it. It is an empty interface. Such interfaces are called as marker interfaces. Other examples of marker interfaces are Cloneable and Remote interfaces.
 
Lets understand serialization and deserializeion of object in following code. Lets take a class Person for this demonstration:
package ot.staff;
import java.io.Serializable;
public class Person implements Serializable 
{
    String strName;
    int age;
    public String getStrName() {
        return strName;
    }
    public void setStrName(String strName) {
        this.strName = strName;
    }
    public int getAge() {
        return age;
    }
    public void setAge(int age) {
        this.age = age;
    }
    public Person(String strName, int age) {
        super();
        this.strName = strName;
        this.age = age;
    }
    @Override
    public String toString() {
        return "Person [strName=" + strName + ", age=" + age + "]";
    }
}
Note that:
  • Here the class Person support serialization by implemeting interface Serializable. Now all objects of class Person support serialization and deserialization.
  • If this is not done, we get an exception java.io.NotSerializableException
ObjectOutputStream oos=null;
try {
    Person p1= new Person("Sachin", 42);
    FileOutputStream fos= new         FileOutputStream("C:\\temp\\myFile.abc");
    oos= new ObjectOutputStream(fos);
    oos.writeObject(p1);
}
catch(Exception ex) {
    System.out.println(ex.toString());
}
finally {
    oos.close();
}

In this code:
  • We are serializing object p1 of class Person in a file. First we create the FileOutputStream for this path. We then wrap this stream into ObjectOutputStream, which provides serialization of objects through a method writeObject(p1);
  • Now the object p1 is actually present in the given file, which we can get back using DeSerialization, as shown below:
ObjectInputStream ois=null;
try {
    Person p1;
    FileInputStream fis= new         FileInputStream("C:\\temp\\myFile.abc");
    ois= new ObjectInputStream(fis);
    p1= (Person)ois.readObject();

    System.out.println(p1);
}
catch(Exception ex) {
    System.out.println(ex.toString());
}
finally {
    ois.close();
}
In this code:
  • We deserialize the object of class Person, back in our code. Note that We use FileInputStream for the given file path, which is then wrapped into  ObjectInputStream, which is used further for de-serialization using method readObject();
  • Importan point to note here is that, we get person object without using new anywhere. We can invoke various methods on this object, just as we do in normal cases of object creation.
 Serialization rules for a class
  1. If a class implements Serializable interface then all its sub classes will also be serializable. So no need to mention it explicitly in sub-classes.
  2. If a class has a reference to another class, all the references must be Serializable otherwise serialization process will not be performed. In such case, NotSerializableException is thrown at runtime.
  3. If there is any static data member in a class, it will not be serialized because static is the part of class not object.
  4. In case of array or collection, all the objects of array or collection must be serializable. If any object is not serialiizable, serialization will fail.
  5. If you don't want to serialize any data member of a class, you can mark it as transient. So data members marked as transient won't be serialized though the object supports serialization.
  6. The serialization process at runtime associates an id with each Serializable class which is known as SerialVersionUID. It is used to verify the sender and receiver of the serialized object. The sender and receiver must be the same. To verify it, SerialVersionUID is used. The sender and receiver must have the same SerialVersionUID, otherwise, InvalidClassException will be thrown when you deserialize the object.
