Wrapper classes
In Java there are two types of data types.
  • Primitive data types: These are built-in data types like int, float, char, float, double etc. These are also called as value-type of data.
  • User defined data types: These are classes we create in our program like Person, Employee, BankAccount etc. These are also called as reference-type of data. 

In many situations, we need to convert the value-type of data into reference-type of data. Wrapper classes help us in doing this conversion.
Primitive Data Types and their Corresponding Wrapper Class:
For every value-type of data, there is corresponding wrapper class, as mentioned below:
Primitive data typeCorresponding Wrapper class
byteByte
shortShort
intInteger
longLong
floatFloat
doubleDouble
boolean Boolean
charCharacter
Need of wrapper classes:
1For passing value type of data by referenceIf we want that a primitive data type passed as argument to a function, should be modified in that function, we need to pass that by reference. However, only reference type of data can be passed by reference. Hence, we need to convert the value type of data to reference type, which is done using wrapper classes.
2For using value type of data in collection framework classes and java.util classesClasses in collection framework and java.util package, accept only reference type of data. So we can't use any of these classes with value type. If we convert these value types into reference types (using wrapper classes), they can very well be used with collection framework classes.
Auto-boxing
In a Java program, if we provide a value type of data where actually a reference type of data is expected, the value type data is automatically converted to reference type (using its corresponding wrapper class). This is called as auto-boxing.
public class Demo {
    public static void main(String[] args) {
        char ch1='Z';
        Character ch2=ch1; /* LHS expects reference type of data but RHS assigns value-type of data. So auto-boxing takes place. */

        List<int> list= new ArrayList<int>(); // Error. Collection classes work only for reference types of data

        List<Integer> list= new ArrayList<Integer>(); /* Ok now. Integer is wrapper class. It is reference type of data */

        list.add(12); /* list object expects Integer. But we have provided 12 which is int. So autoboxing takes place here */
    } 
}
Un-boxing
This is exactly opposite of auto-boxing. In a Java program, if we provide a reference type of data where actually a value type of data is expected, the reference type data is automatically converted to value type (using its corresponding wrapper class). This is called as un-boxing.
public static void main(String[] args) {
        Character ch1='Z';
        char ch2=ch1; /*LHS expects value type of data but RHS assigns reference-type of data. So un-boxing takes place.*/
        List<Integer> list= new ArrayList<Integer>(); 
        list.add(12); 
        int a= list.get(0); /*RHS returns reference type of data but LHS expects value type of data. So un-boxing takes place.*/
    }
