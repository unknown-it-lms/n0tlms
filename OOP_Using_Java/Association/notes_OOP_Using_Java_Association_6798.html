Association
For modeling some entities, we need to create object of one class as data member of another class. In such situation, the two classes are considered to be associate with each other. Association creates HAS-A relationship between the two classes.

Association relationship can be of following types:
  • One-to-one association: One object has only one object of other class as its data member.
  • One-to-many association: One object has many objects of other class as its data member.
  • Many-to-one relationship: Many objects share one object between them as their data member
  • Many-to-many relationship: Many objects have one or more objects of other class as their data member

Following are some of the examples of associations.
Person HAS date of birthObject of Date class is created as data member of Person class to model date of birth of the person.One person has only one data of birth so this is One to one relationship
Car HAS engineObject of Engine class is created as data member of Car class.One car has only one engine so this is One to one relationship 
Line HAS two pointsTwo objects of Point class are created as data members of Line class for modeling first point and second point of a line. One line has two points. So this is one-to-many relationship
Triangle has three pointsThree objects of Point class are created as data members of Triangle class for modeling first point, second point and third point of a triangle. One triangle has three points. So this is one-to-many relationship
One person can have local address as well as permanent addressTwo objects of Address class are created as data members of Person class to model these two addresses. This is one-to-many relationship
Many batsman face many bowler in a cricket match Many objects of Bowler class gets associated with many objects of batsman class.This is many-to-many relationship
Creating association relationship
Association relationship is modeled in Java by following steps:
  1. Create reference of second class as data member of first class
  2. Initialize the reference of second class in the constructor of first class
  3. Use the functionalities of second class for modeling the behavior of first class. This is also called as delegation.

Lets model the association between Person and date of birth by following steps given above.
Person.javaDate.java
package Utility;
public class Person {
    String strName;
    Date dob; /* 1.Reference of Date class created inside person class for modeling data of birth of the person. Note that this is just a reference which needs to be initialized later */

    public Person() {
        strName="";
        dob= new Date(); /* 2. Here reference dob is initialized by creating date object using default constructor. */
    }
    public Person(String str, int a,int b, int c) {
        strName=str;
        dob= new Date(a,b,c); /* 3. Here reference dob is initialized by creating date object using parameterized constructor. Person constructor received all the data needed for creating person object. It uses str for initializing the name of person and remaining data d, m, y is used for initializing the data of birth of person. */
    }
    public String Display() {
        return strName+" "+dob.Display(); /* Display of person delegates part of the responsibility to dob object. This is called as delegation of responsibility. This way we use existing entity for creating another entity. */
    }
}		package Utility;
public class Date {
    int d,m,y;

    public Date() {
        d=m=y=0;
    }
    public Date(int a,int b,int c) {
        d=a;
        m=b;
        y=c;
    }
    public String Display() {
        return d+"/"+m+"/"+y;
    }
}
Test.javaOutput:
import Utility.Person;
public class Test {
    public static void main(String[] args) {
        Person p1;
        p1 = new Person("Sachin", 24, 4, 1973); /* All data is passed to Person constructor which distributes it appropriately between itself and date object */
        String str= p1.Display(); /*Person's Display delegates part of the responsibility to Date's Display() */
        System.out.println(str);
    }
}		Sachin 24/4/1973

Note that part of the output (Sachin) is generated by class Person and remaining part of the output (24/4/1973) is generated by class Date.
In some cases, the object of second class is created outside and simply passed as parameter to constructor of the first class. This can be done as given below:

package Utility;
public class Person {
    String strName;
    Date dob; 

    public Person() {

        dob= null;
    }

Types of associations
Association relationship is further classified into two types:
  1. Aggregation
  2. Composition

AggregationIn this type of association, both the objects can exist independently.
For example
  • Every car has a driver. But a car can exist without a driver and a driver can exist without the car. 
  • Students have a trainer. But students can exist with a trainer and trainer can exist without the students.
This relationship is also called as weak-association. This is truly the has-a relationship
CompositionIn this type of association, both the objects can NOT exist independently. When the outer object gets deleted, the inner should also be deleted.
For example
  • Every car has an engine. But a car can NOT exist without an engine and a engine can NOT exist without the car. When a car is destroyed, engine also gets destroyed automatically. 
  • Exam has schedule. But schedule can NOT exist with an exam and exam can NOT exist without the schedule. When a exam is canceled, its schedule also gets canceled automatically.
This relationship is also called as strong-association. This is also called as part-of relationship.
