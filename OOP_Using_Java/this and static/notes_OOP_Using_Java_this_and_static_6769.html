this and static
Lets now understand the concepts of this reference and static keyword in Java.
this reference
  • "this" is an hidden local variable created by the compiler in every member function of a class. It is actually a reference that refers to the object that has called the function.
  • It is automatically created by the compiler, in every non-static member function of a class.
  • It is a local variable of that member function. It gets created automatically on the stack when the function is called and gets removed from the stack when the function returns.
Consider class Date
class Date {
    int d, m, y;
    public void getDay() {
        //  this reference refers to invoking object.
        // Here, its data type is Date reference
    }
}
class Test {
    public static void main(String[] args) {
    Date d1= new Date ();
    Date d2= new Date ();
    d1.getDay(); // In this function call, this refers to d1
    d2.getDay(); // In this function call, this refers to d2
    }
}
Now consider class Person
class Person {
    String name;
    int age;
    public void getName() {
        //  this reference refers to invoking object.
        // Here, its data type is Person reference
    }
}
class Test {
    public static void main(String[] args) {
    Person p1= new Person ();
    Person p2= new Person ();
    p1.getName(); // In this function call, this refers to p1
    p2.getName(); // In this function call, this refers to p2
    }
}

Use of this reference
"this" reference is used for following purposes in Java program.
1To avoid masking of data member by a local variable.

class Date {
    int d, m, y;

    // Note that here parameter names are same as names of data members
    public Date(int d, int m, int y) {

        d=d; /*Wrong: Parameter being a local variable, it gets assigned to itself. This is called as self-assignment. Our intention is to assign value of parameter d to data member d, which fails due to same name. This is called as variable masking or variable shadowing. */

    /* To avoid this, we specify explicitly that local variable d should be assigned to data member d, as follows */
    this.d=d; // Now data member d gets initialized with parameter
    }
}
2To call once constructor from another of same class

class Date {
    int d, m, y;

    public Date(int d, int m, int y) { 
        // Do the validation of values of day, month and year. If valid, assign else don't
        this.d=d;
        this.m=m;
        this.y=y;
    }
    public Date(int d,int m) {
        this(d,m,0); // Two-int constructor calls parameterized constructor using this
    }
}
3To return invoking object from a member function

class Date {
    int d, m, y;
    
    Date someFunction() {
        return this; // Invoking object is returned from this function
    }
}
Static data
  • Static keyword is used to create a data member that is common for all objects of that class. 
  • Non-static data is created separately for all each instance. Hence, they are also called as instance variables.
  • This is used to stored some information is guaranteed to have same value for all instance of that class. However, note that the value of static data member CAN be changed. Just one copy of that variable will be shared by all instances of that class.
class Date {
    int d, m, y; /* Non-static data members of the class. Created separately for each instance of date */

    static int count; /* Static data member of class. Created only once and shared by all instances of date */
}
class Test {
    public static void main(String[] args) {
        Date d1= new Date();
        Date d2= new Date();
        /* Note that data members d, m and y are created for d1 and d2 separately. However, count, being static, gets created only once and is shared by d1 as well as d2. */
    }
}
Example of static data member:
  • Count of objects for various classes
  • Rate of interest for bank account.
  • Registration fees for all bikes
  • Exam schedule for all students of a class 
 
Static function
  • Static functions are the member functions of a class that can be called without using object of that class. They are called using just the name of that class.
  • As static functions don't have invoking object, they don't have "this" reference.
  • Static functions can access ONLY static data members. However, non-static functions can access static as well as non-static data members.

class Date {
    int d, m, y; /* Non-static data members.*/
    static int count; /* Static data member */
   
    // constructor is a non-static member function
    public Date() {
        d=m=y=0; // Can access non-static data 
        count++; // Can access static data also
        this.d=10; // Can use this reference
    }

     // This is a static member function.
    static int getCount() {
        d=0; // ERROR. Can't access non-static data member
        return this.count; // ERROR. Don't have this reference
        return count; // Can access static data member
    }
}		class Test {
    public static void main(String[] args) {
        Date d1= new Date();
        Date d2= new Date();
        /* static functions can be called using class name */
        int iCountOfObjects= Date.getCount();
       }
}
Static constuctor
Java doesn't allow use of static keyword for a constructor. It gives compile-time error as shown below.
class Person {
    String name;
    int age;
    public static Person() { /* ERROR: Illegal modifier for the constructor in type Person; only public, protected & private are permitted */
    }
}
If we want to initialize static data members of a class, we should use static code blocks. 
Static code blocks
  • Static code block can be used to initialize static data members of a class. 
  • They are executed when the class is loaded by the classloader.
  • Static code block are always public by default. We are not allowed to mention public keyword for them.
  • One class can have one or more static code blocks. All get executed one by one from top to bottom order.
  • Static code blocks don't have "this" reference. Hence, they can access only static data member but can't access non-static data members.
Date.javaTest.java
class Person {
    String name;
    int age;
    static int count;
    static  { // This is static code block. 
        count=0;
    }
     public static  { /* ERROR.  Don't mention public. They are ALWAYS public by default. */
        count=0;
    }
    static int getCount() {
        return count;
    }
}		class Test {
    public static void main(String[] args) {
          Person p1; // Class Person not loaded here
          p1= new Person(); // Class Person gets loaded now
          Person p2 = new Person(); /* Class Person already loaded while creating first object of Person. Hence, it is not loaded again. */
        Person.getCount(); /* This will also load the class, if not already loaded. */
       }
}
-----------------------------------------------
Note that class is NOT loaded if we just create a reference of that class. It gets loaded when 
  • We create first instance of that class OR
  • We call a static member function of that class
One class can have more than one static code block also. All get executed from top to botton when the class is loaded.
class Person {
    String name;
    int age;
    static int count;

    static { // Static block number 1
        count=0;
    }
    static  { // Static block number 2
        count=1;
    }
    public Person() {
        name="Not set";
        age=0;
    }
    public Person(String str) {
        name=str;
        age=0;
    }
    static { // Static block number 3
        count=2;
    }
    static int getCount() {
        return count;
    }
    static { // Static block number 4
        count=3;
    }
}
class Test {
    public static void main(String[] args) {
    int n = Person.getCount();
    System.out.println("Value of count: "+n);
    }
}
Gives output: 
Value of count: 3 
------------------------------------
Note that: 
All static code blocks get executed from top to bottom when the class gets loaded. Hence the final value assigned to count is 3
