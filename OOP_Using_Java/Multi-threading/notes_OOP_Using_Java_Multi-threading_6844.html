Multi-threading
Now as we are aware of the basic programming needs, handling runtime error, using IO library, we will move further for learning very important API in java library. This API is used for creating multi-threaded applications which is very important for real life applications. As java's purpose is to execute itself in networked, distributed environment, using multi-threading becomes very important. In this chapter we will learn about what is a thread, creating threads and interrupting the thread by suing sleep and join method.
Concept of thread
​​​​​​​Need of multithreading :
Threading is a facility to allow multiple activities to run parallely within a single process. Most modern operating systems support threads. Java is the first mainstream programming language to explicitly include threading within the language itself, rather than treating threading as a facility of the underlying operating system.
From the beginning, purpose of java applications is to execute in a networked and distributed environment. Application may get halted because of some reason like waiting for user input, waiting for some response from remote machine etc. In these scenarios CPU remains idle. Multithreading helps in simultaneous execution of two or more parts of a program to utilize the CPU time maximumally. A multithreaded program contains two or more parts that can run concurrently. This way, using multi-threading, Idle time of CPU can be kept to minimum. This approach of creating and executing multiple threads simultaneously is known as multithreading.
Benefits of multithreading :
  1. Better use of system resources.
  2. Parallel execution of tasks and thus less execution time
  3. Enhanced performance on multi-processor machines
  4. Improved GUI responsiveness
  5. Independent threads (If an exception occurs in one thread, it won't affect execution of other threads)
Concept of thread :
  1. Threads are lightweight sub-processes,
  2. The thread is the smallest unit of execution which has its separate path of execution
  3. Multiple threads within a process share the same memory address space, which means they have access to the same variables and objects, and they allocate objects from the same heap
Note : Thread is nothing but finally a chunk of code which is assigned some task
Thread Lifecycle
The thread goes through different states right from when it is created upto it finishes it's task.
​
States through which the thread has to pass to complete its lifespan.
NewA thread is in New state when you create the instance of thread class, but the start() method is yet to get called.
RunnableThe thread is in runnable after executing the start() method. In this stage, it waits for the thread scheduler to run it.
RunningA thread picked by the thread scheduler for execution remains in running state. Actual CPU time is allocated to the thread
Non-Runnable (blocked)In this state, the thread remains alive, but it is not eligible to run. It may be due to some sleep operation, waiting for any File I/O to finish or in the locked state, etc.
TerminatedA thread is said to be in the terminated state when it?s Run() method exits which indicates that task of the thread has been completed
Note : In the above diagram focus on arrows. From runnable state thread will never come back to new state. Similarly thread will not enter from new to running state directly.
Creating threads
There are two ways to create a thread in Java:
  1. By extending Thread class
  2. By implementing Runnable interface

Lets see each of them in details:
1. By extending Thread class
public class MyThread extends Thread
{
    public void run()
    {
        for(int i=1;i<=30;i++)
        {
            System.out.println("Child thread: "+i);
        }
    }
}		public class MyThreadDemo
{
    public static void main(String[] args)
    {
        Thread t = new MyThread(); /*creating thread instance, in new state */
        t.start(); /* thread converted to runnable state */
        for(int i=1;i<=30;i++)
        {
            System.out.println("Main thread: "+i);
       }
    }
}
In the above code, main thread creates child thread and then both the threads run concurrently. 
Important methods of Thread class:
getName()It is used for Obtaining a thread's name
getPriority()Obtain a thread's priority
isAlive()Determine if a thread is still running
join()Wait for a thread to terminate
run()Entry point for the thread
sleep()suspend a thread for a period of time
start()start a thread by calling its run() method

Here, run is the method which is used to start the thread. All other methods are used for controling the thread execution. 
2. By implementing Runnable interface
public class MyTask implements Runnable
{
    public void run()
    {
        for(int i=1;i<=30;i++)
        {
            System.out.println("child thread prints : "+i);
        }
    }
}		public class MyTaskDemo
{
    public static void main(String[] args)
    {
        MyTask task = new MyTask();
        Thread th = new Thread(task);    //creating a thread by one more constructor
        th.start();
        for(int i=1;i<=30;i++)
        {
            System.out.println("main thread prints : "+i);
        }
    }
}
The Runnable interface is single abstract method interface. It has only one method run() which is used to define the task to be performed by the thread. For other controlling methods of thread, Runnable depends on Thread. Note: Design guidelines suggest, if no other method than the run() method in the thread class is reimplemented, it is better to create the thread by implementing Runnable interface.
Default thread
  • When a Java program starts up, one thread begins running immediately. This is usually called the main thread of our program, because it is the one that is executed when our program begins.
  • It is the thread from which other child threads will be spawned (created).
  • Often, it must be the last thread to finish execution because it performs various shutdown actions
  • The main thread is created automatically when our program is started. To control it we must obtain a reference to it. This can be done by calling the method  currentThread()  which is present in Thread class. 
    Thread cur_thread = Thread.currentThread();      //static method in Thread class
  • This method returns a reference to the thread on which it is called. The default priority of Main thread is 5 and for all remaining user threads priority will be inherited from parent to child.
Types of threads:
Threads are of two types:
  1. User thread: These type of thread or Non-Daemon are designed to do specific or complex task where as daemon threads are used to perform supporting tasks. JVM doesn't wait for daemon thread to finish but it waits for User Thread.
  2. Daemon thread:  These type of threads are low priority threads which always run in background and user threads are high priority threads which always run in foreground.
Use of Sleep, Join
​​​​​​​Many times we need to interrupt a running thread. sleep() and join() both the methods are used for thread interruption. Lets see them in detail now:
1. sleep() method
public class TableThread extends Thread {
    int num;    
    public TableThread(int num) {
        super();
        this.num = num;
    }
    public void run() {
        for(int i=1;i<=10;i++) {
            System.out.println(num + "*" + i + " = " + (num*i));
            try {
                Thread.sleep(500);
            }
            catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}		This thread prints the table of a specified number by sleeping for 500 ms before every next iteration.

Note : This method throws InterruptedException which is checked exception so it needs to be handled or declared using throws clause
Note that: 
  • Thread.sleep causes the current thread to suspend execution for a specified period. This is an efficient means of making processor time available to the other threads of an application or other applications that might be running on a computer system.
  • Thread.sleep() method needs the time specified in milliseconds. The argument value for milliseconds can't be negative, else it throws IllegalArgumentException.
  • One more overloaded sleep version is available which accepts time in miliseconds and nanoseconds
  • The actual time thread sleeps before waking up and start execution depends on system timers and schedulers. For a quiet system, the actual time for sleep is near to the specified sleep time but for a busy system it will be little bit more.
  • Thread sleep doesn't release the lock on current thread

2. join() method
    JoinTest t1 = new JoinTest();
    JoinTest t2 = new JoinTest();
    JoinTest t3 = new JoinTest();
    t1.start();
    try {
         t1.join(); /* calling join() method */
      }
    catch(Exception e)
    {
        System.out.println(e);
    }
    t2.start();
    t3.start();		In this case join() method is called on t1 by main thread. So main thread will not start running until t1 thread finishes.
Note that:
  • The join() method is used to hold the execution of currently running thread until the specified thread is dead. (finished execution).
  • A join() is a final method of Thread class
  • It can even be used to join the start of a thread's execution to the end of another thread's execution so that a thread will not start running until another thread has ended.
  • join() method has got overloaded versions which accepts even the time in miliseconds.

Note that:
join() and sleep() both the methods throw InterruptedException.
sleep() is a static method in Thread class whereas join() is a non-static method.
Thread synchronization
Till now, we have seen how to create threads and run them parallely with other threads. However, in real scenario, responsibility of a programmer doesn't end here. In many situations, multiple threads running parallely may corrupt some data or lead to errors. Hence, we need to ensure that the thread run in a well coordinated manner to avoid such errors.

This task generally falls into following two categories:
  • Ensure mutual exclusion
  • Inter-thread communication
Ensure mutual exclusion
There are some tasks such that only one thread should execute that task at a time. If a thread is already running that task, the next thread should start executing that task only when the previous thread finished that task completely. In short, we should ensure that only one thread is running that code at a time. This can be achieved using thread synchronization techniques. There are three ways in which threads can be synchronized.
  1. Method level Synchronization
  2. Block Level Synchronization
  3. Static Synchronization 

Lets see each of them now:
1. Method level Synchronization
void synchronized print() // method is synchronized    
{      
        
 }		Here, the method print is synchronized. This ensures that if a thread is already executing this method, other threads won't be able to execute it till the first thread returns from this method.
2. Block Level Synchronization
void print() // method is NOT synchronized    
{      
        // code 1
        synchronized(this)
        {
            // code 2
        }
        // code 3
 }		Here, instead of synchronising the entire method, only that part of the code which needs mutual exclusion, is synchronized. 
Here, as code 1 and code 3 are outside the synchronized block, they can run parallely in multiple threads. But, as code 2 is inside a synchronized block, only one thread can execute it at a time.
This approach gives better performance without compromising on the synchronization.
3. Static Synchronization :
In the above two approaches, the invoking object is used as lock. This will work fine as long as same object is used in multiple threads. But, if different theads use different objects, then both the above methods fail to achieve synchronization because every thread will lock different object. 
To handle such situations, we can have static synchronization, where are static method is Synchronization. As the method is static, it uses the Class object as lock. Hence, same object will be used as lock in different threads. This is done as follows:
class MyPrinter {
    synchronized static void printTable(int n)
    {
        // Code    
    }
 }
}		The method printTable is static synchronized. It uses the class object of MyPrinter class as lock.
Inter-thread communication
This is used in scenarios where at some point, one thread needs to wait for some other thread to perform some task. So the first thread should wait till the second thread notifies it. 
This is achieved using the methods wait(), notify(), and notifyall():
wait()This is method is called by a thread which want to wait for a signal (notification) from other thread.
notify()The thread which is responsible for giving the notification calls notify() method so that the waiting thread gets a notification and can continue its execution with those updated items. This method notifies only one waiting thread.
notifyall()This is same as notify() except that it notifies ALL waiting threads.
