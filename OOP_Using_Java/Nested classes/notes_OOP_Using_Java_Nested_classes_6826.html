Nested classes
  • We have seen till now that a class in Java can have data as well as functions as its members. The data present in a class is called as data member and the functions present inside a class are called as member functions.
  • Even a class can be declared as a member of another class. Such classes are called as nested classes.
  • Just as other members, a nested class can be either static member or a non-static member of the outer class. Based on this, there are two types of nested classes:

  1. Static class (The nested class is static member of the outer class)
  2. Inner Class (The nested class is non-static member of the outer class)
static class
​​​​​​​A class declared as a static member of other class is called as static class.
class A { // Outer class
    int a1;
    public int getA1() {
        return a1;
    }
    static class B { // static class
        int b1;
        public int getB1() {
            return b1;
        }
    }
}		public class Test {
    public static void main(String[] args) {
        // use the outer class as usual
        A a1= new A();
        int x= a1.getA1();
        System.out.println(x);
        
        /*Using static class here. Note that we must access the static class  using the name of outer class, just the way we access static member of a class*/
        A.B objB= new A.B();
        int y= objB.getB1();
        System.out.println(y);
    }
}
A static class can access only the static members of outer class. It can't access non-static member of outer class directly.class A {
    static int aCount;
    int a1;
    public int getA1() {
        return a1;
    }
    static class B {
        int b1;
        public void setOuterCount(int val) {
            aCount=val; /* OK. Functions of a Static class can access static data of outer class directly. */
        }
        public int getB1() {
            return a1+b1; /*Error: Non-static data of outer class can't be accessed in functions of static class */
        }
    }
}
inner class
A class declared as a non-static member of outer class is called as inner class.
class A { // Outer class
    static int aCount;
    int a1=10;
    public int getA1() {
        return a1;
    }
    // Non-static nested class called as Inner class
     class B {
        int b1=20;
        public int getB1() {
            return b1;
        }
    }
}		public class Test {
    public static void main(String[] args) {
        // Use the outer class as usual
        A a1= new A();
        int y= a1.getA1();
        System.out.println(y);

        /*For accessing the inner class, we need object of outer class. Reference is created as A.B objB but object is created using object of outer class, a1 */
        A.B objB=a1.new B();
        int x= objB.getB1();
        System.out.println(x);
    }
}
Inner class can access static as well as non-static members of outer class. class A {
    static int aCount;
    int a1;
    public int getA1() {
        return a1;
    }
    class B {
        int b1;
        public void setOuterCount(int val) {
            aCount=val; /* OK. Functions of a Inner class can access static data of outer class directly. */
        }
        public int getB1() {
            return a1+b1; /* OK. Functions of a Inner class can access non-static data of outer class directly. */
        }
    }
}
Anonymous inner class
Many times we write a class for extending a class or for implementing an interface. If this class is used only once, then we need not give any name to that class. Such class, created without name, either for extending existing class OR for implemeting an interface, is called as anonymous inner class. 

Lets see both the types now.
  1. Anonymous class for extending a base class
  2. Anonymous class for implementing an interface.

1. Anonymous class for extending a base class
Consider the Date class we have already written. If we want to change the display format of toString() without modifying the class Date, we have two options. Either extend a class MyDate and override the method toString() OR use anonymous class. Lets see both of them.
By extending class MyDate from class DateUsing anonymous class for extending class Date
class MyDate extends Date {
    MyDate(int d,int m,int y) {
        super(d,m,y);
    }
    public String toString() {
        return String.format("%d-%d-%d", d,m,y);
    }
}
public class Test {
    public static void main(String[] args) {
        Date d1= new MyDate(12,3,2009);
        System.out.println(d1);
    }
}
public class Test {
    public static void main(String[] args) {
/* Here we have created an anonymous class by extending the class Date, created its instance and used its reference as well */     
        Date d2= new Date(3,4,2009) {
            public String toString() {
                return String.format("%d/%d/%d", d,m,y);
            } // No semicolon here..
        }; // Give semicolon here...
        System.out.println(d2);
    }
}
2. Anonymous class for implementing an interface
Let iCalculator be an interface we want to implement. We can do it in two ways. Either by writing a class MyCalculator that implements the interface iCalculator OR by using an anonymous class for doing the same.
By writing a new class MyCalculator for implementing iCalculator .Implementing iCalculator in an anonymous class.

interface iCalculator{
    public int Add(int a,int b);
    public int Subtract(int a,int b);
}

class MyCalculator implements iCalculator{
    public int Add(int a,int b) {
        return a+b;
    }
    public int Subtract(int a,int b) {
        return a-b;
    }
}
public class Test {
    public static void main(String[] args) {
        iCalculator cal= new MyCalculator();
        System.out.println(cal.Add(12, 13));
        System.out.println(cal.Subtract(12, 13));
    }
}
interface iCalculator{
    public int Add(int a,int b);
    public int Subtract(int a,int b);
}
public class Test {
    public static void main(String[] args) {
/* Here we have created an anonymous class for implemeting the interface iCalculator, created its instance and used its reference as well */         
        iCalculator cal= new iCalculator() {
            public int Add(int a, int b) {
                return a+b;
            }
            public int Subtract(int a, int b) {
                return a-b;
            }
        };

        System.out.println(cal.Add(12, 13));
        System.out.println(cal.Subtract(12, 13));
    }
}

Lambda expression
  • Lambda expressions are used for implementation of functional interaces. 
  • Functional interfaces is the interface with only one abstract method. 
  • Lamda expression implements the functional interface without creating a class as well as without even mentioning the method name. As their is only one abstract method in functional inteface, it is understood that the implemention is only for that method.
interface iEmiCalculator{
    public double calculateEmi(double amount, double years, double rateOfInt);
}
  • iEmiCalculator is a functional interface. 
  • It has only one method that accepts loan amount, number of year and rate of interest as parameter. The implementing class is expected to return the emi for this loan
public class Test {
    public static void main(String[] args) {
        iEmiCalculator cal= (a,y,r)->{return (a*y*r/100);};
        System.out.println(cal.calculateEmi(12000,3,4.8));
    }
}
  • We have used an lambda expression that implements the iEmiCalculator interface (because the reference used on LHS is iEmiCalculator).
  • It writes a method without name but with signature that matches the only method in the interface iEmiCalculator.
Lambda expressions can be used as a predicate passed to other functions as given below:
// This is a funnctional interface.
interface iProcesData {
    int Process(int a);
}
public class Test {

    public static void main(String[] args) {
        int [ ]arr= new int[] {10,20,30,40,50};
        // Lambda exp to implement functional interace
        iProcesData process1=(a)-> {return ++a;};
        DoProcess(arr,process1);
        System.out.println();

        /* Lambda exp to implement functional interace in the function call itself. */
        DoProcess(arr,(a)-> {return --a;});
    }
    public static void DoProcess(int [] arr,iProcesData p) {
        for(int a:arr) {
            System.out.println(p.Process(a));
        }
    }
}		Output:

Benefits of using Lamda expressions:
  1. They are the shortest possible implementation of a functional interface. 
  2. Once we understand their syntax, they make the code more readable.
  3. The function cal be treated as an object and passed to other functions in the same way as we pass data like int, float or strings.
