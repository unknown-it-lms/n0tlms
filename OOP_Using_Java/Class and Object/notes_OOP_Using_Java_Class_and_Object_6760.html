Class and Object
The Object oriented programming we have seen earlier, recommends modelling of entities in our software system. It suggests the principles of Abstraction and Encapsulation for this purpose.
  • Class is the concept in Java that implements principles of Abstraction and Encapsulation and creates an object oriented user-defined data type. This data type models real-life entities in software system.
  • Objects are the variables (called as instances) of this data type which can be used in our program further.

Note that Class is like a factory that defined structure and behaviour of an entity and objects are the products created from that factory. As mentioned earlier, Class is an object-oriented user-defined data type, just like a built-in data type but with following differences and simillarities.

constructor
Constructor is a member function of a class which is used to initialize an object at the time of creation itself.
class Date {
    int d, m, y;

    // This is constructor
    public Date() {
    d=m=y=0
    }
}		It has following features.
  1. It's name must be same as that of the class.
  2. It should not return anything. Don't write even void as its return type.
  3. It is a function which we write in the class but called by compiler immediately after creating an object for its initialization.
--------------------------------------------------------------------------------------------
Note that constructor is used to initialize an object. It doesn't create an object. Objects are created by programmer using new operator.

Overloading constructor
​​​​​​​We can write more than one function in a class, each for initializing the object in a specific way. Appropriate constructor is called by the compiler based on the parameters passed for initialization of object.
Person.javaTest.java

    // This is Paramter-less constructor. Also called as        //  default constructor

Default constructor
  • If we don't write any constructor in our class, compiler provides a default constructor. This constructor has a do-nothing definition. This is called as compiler's default constructor.
  • However, note that compiler provides this facility only for those classes which don't have ANY constructor. If any class has even one constructor of any type, this facility is not provided by compiler for that class.
Class without a constructorTest.java
class Person {
