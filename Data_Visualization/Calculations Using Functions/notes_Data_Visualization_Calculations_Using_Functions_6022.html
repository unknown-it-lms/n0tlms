Calculations Using Functions
​​​​​​​Tableau is a powerful data visualization tool that allows users to connect, analyze, and visualize data. To create insightful visualizations, Tableau provides a wide range of calculations that help you manipulate and transform data.
Tableau Documentation: Get Started with Calculations in Tableau​​​​​​​

In Tableau, a calculated field is a custom expression that you create using existing data fields. It allows you to perform calculations and manipulate data in ways that are not possible using the original fields alone. Calculated fields can be used to generate new insights, create new metrics, and build complex analysis without altering the source data.

Creating a calculated field involves writing a formula using Tableau's formula language. The formula language is similar to Excel or SQL expressions, and it includes a wide range of functions, operators, and constants to perform calculations on the data.

To create a calculated field in Tableau, follow these steps:

  1. In the Tableau data pane, right-click on the data source, a specific field, or a blank area, then select "Create Calculated Field."

    ​

  2. The calculated field editor will appear, where you can write your formula. Give the calculated field a meaningful name.

    ​

  3. Use the formula language to write the calculation. The language supports basic arithmetic operations (addition, subtraction, multiplication, division), mathematical functions (e.g., SIN, COS, AVG), logical functions (e.g., IF, AND, OR), string functions, and more.

  4. After writing the formula, click on "OK" to create the calculated field.

Once the calculated field is created, you can use it like any other field in Tableau. You can add it to the rows or columns shelf, use it in filters, combine it with other calculated fields, or use it in visualizations.

Some common use cases for calculated fields include:

  1. Aggregations: Creating a new measure by performing aggregation functions like sum, average, count, etc., on existing data fields.

  2. Data Transformations: Applying mathematical or logical operations to modify the data, such as converting units, normalizing values, or rounding numbers.

  3. Conditional Logic: Using IF statements to define conditions and create new categories or flags based on the data.

  4. String Manipulation: Concatenating strings, extracting substrings, or formatting text.

  5. Date Calculations: Calculating date differences, extracting parts of dates, or generating date-based metrics like month-to-date sales.

Calculated fields in Tableau are powerful tools that allow analysts and data professionals to derive valuable insights and answer complex questions from their data without requiring changes to the original dataset.

Numeric, String & Date Calculations
​​​​​​​Numeric, string, and date calculations are common types of calculations used in data analysis and reporting. Each type of calculation serves a different purpose and involves specific operations or functions. 
  1. Numeric Calculations: Numeric calculations involve performing mathematical operations on numerical data. Common numeric calculations include:
  • Arithmetic Operations: Addition (+), Subtraction (-), Multiplication (*), Division (/)
  • Aggregation Functions: SUM, AVG (average), MIN (minimum), MAX (maximum), COUNT (count of values)
  • Percentage Calculations: Calculating percentages, percentage changes, or shares of a whole.

  1. String Calculations: String calculations involve manipulating and transforming text data. Common string calculations include:
  • Concatenation: Combining two or more strings together.
  • Substring: Extracting a part of a string based on position or a specific character.
  • Length: Calculating the number of characters in a string.
  • Replacing: Replacing specific characters or substrings within a string.

  1. Date Calculations: Date calculations involve working with date and time data. Common date calculations include:
  • Date Differences: Calculating the difference between two dates in days, months, or years.
  • Date Aggregations: Grouping data by date components (e.g., day, month, year) for analysis.
  • Date Manipulation: Adding or subtracting days, months, or years from a date.


Level of Details (LOD) Expressions
​​​​​​​In Tableau, LOD (Level of Detail) expressions are powerful and flexible calculations that allow you to compute values at different levels of granularity than the visualization's current level. They provide a way to perform aggregations or calculations that are independent of the visualization's dimensions. LOD expressions are especially useful when you need to create complex calculations that involve aggregations across different levels of detail.

Tableau Documentation: Create Level of Detail Expressions in Tableau

1. Fixed LOD Expression:
The FIXED LOD expression allows you to calculate an aggregation at a specific level of dimensions in the view, regardless of what dimensions are included in the visualization. It freezes the aggregation to a defined level of granularity.
Syntax:
                                  ​​​​​​​{ FIXED [Dimension1], [Dimension2], ... : AggregationFunction(Measure) }
Example:
Suppose you have a dataset with sales data and you want to calculate the average sales for each product category, regardless of any filters or dimensions applied to the visualization:
                                                                 { FIXED [Product Category] : AVG([Sales]) }

2. Include LOD Expression:
The INCLUDE LOD expression allows you to calculate an aggregation at a specific level of dimensions while considering the dimensions present in the visualization. It includes specified dimensions in the calculation while keeping others unaffected.

Syntax:
                                  { INCLUDE [Dimension1], [Dimension2], ... : AggregationFunction(Measure) }
Example:
Let's say you have a dataset with sales data, and you want to calculate the sum of sales for each product sub-category while considering the filters applied to the visualization but keeping the product category unaffected:
                                                             { INCLUDE [Product Sub-Category] : SUM([Sales]) }

3. Exclude LOD Expression:
The EXCLUDE LOD expression allows you to calculate an aggregation at a specific level of dimensions, excluding certain dimensions from the calculation. It excludes specified dimensions from the aggregation while keeping others unaffected.
Syntax:
                                 { EXCLUDE [Dimension1], [Dimension2], ... : AggregationFunction(Measure) }
Example:
Suppose you have a dataset with sales data, and you want to calculate the average sales per customer, excluding the product category from the calculation:
                                                              { EXCLUDE [Product Category] : AVG([Sales]) }
Steps to Create LOD:
Step 1: Connect to Data
Connect Tableau to your data source and drag the relevant data to the workspace. In this example, we assume you have a dataset with columns "Product Category" and "Sales."

Step 2: Create a New Worksheet Click on the "New Worksheet" tab at the bottom of the Tableau window to create a new worksheet where we will perform the LOD calculation.

Step 3: Open the Calculated Field Editor Right-click in the "Data" pane on the left side, then select "Create Calculated Field."

Step 4: Write the FIXED LOD
Expression In the calculated field editor, give your calculated field a name, such as "Average Sales per Category," and write the FIXED LOD expression.
​

Explanation of the expression:

  • The curly braces {} indicate that it's an LOD expression.
  • "FIXED" specifies that we want to fix the aggregation to a specific level of detail.
  • [Product Category] is the dimension we want to calculate the average sales for.
  • "AVG([Sales])" is the aggregation function to calculate the average sales.

Step 5: Click "OK"
Once you have written the expression, click "OK" to create the calculated field.

Step 6: Use the Calculated Field in Visualization
Drag and drop the newly created calculated field, "Average Sales per Category," to the Rows or Columns shelf, depending on how you want to display the data.

Step 7: Observe the Results
You should now see the visualization with the average sales for each product category, regardless of any filters or dimensions applied to the view. The aggregation remains fixed at the product category level.

Note: LOD expressions can be used in various scenarios, and the syntax may differ depending on the type of LOD expression you want to create (FIXED, INCLUDE, or EXCLUDE). Always ensure that you select the appropriate type of LOD expression based on your analysis needs.
