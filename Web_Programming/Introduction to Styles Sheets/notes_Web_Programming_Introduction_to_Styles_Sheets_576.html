Introduction to Styles Sheets
Understanding of HTML is really incomplete unless we learn fundamentals of CSS. CSS learning is very imprtant because it gives nice and consistent look and feel to web pages in the web application. Styling information can be written in various ways. We will understand all these different ways of style sheets in this chapter.
Need of Style sheets
CSS stands for Cascading Style Sheets and almost all webpage's written with HTML use it.
CSS is a style sheet language that adds styling and formatting to web pages. The purpose of CSS is to provide a standard way to define, apply, and manage sets of style rules. 
The "cascading" in Cascading Style Sheets refers to how styling values are applied in the context of the parent/child hierarchy of the Web page. Child elements either inherit or override styling bound to their parent elements.
Advantages of CSS 
  1. Easy maintenance : To make a global change, simply change the style, and all elements in all the web pages will be updated automatically.
  2. Better styles : CSS has a much wider array of attributes than HTML, so you can give a far better look to your HTML page in comparison to HTML attributes.
  3. Multiple Device Compatibility : Style sheets allow content to be optimized for more than one type of device. By using the same HTML document, different versions of a website can be presented for handheld devices such as PDAs and cell phones.
  4. Global web standards : Now HTML attributes are being deprecated and it is being recommended to use CSS. So its a good idea to start using CSS in all the HTML pages to make them compatible to future browsers. e.g border attribute of the table is replaced by corrosponding CSS attributes.
  5. CSS saves time : You can write CSS once and then reuse same sheet in multiple HTML pages. You can define a style for each HTML element and apply it to as many Web pages as you want.


Types of Style Sheets
There are three ways of writing a style sheet:
  1. External style sheet
  2. Internal style sheet
  3. Inline style
External style sheet
An external style sheet is a separate file containing CSS (Cascading Style Sheets) rules that define the styles to be applied for different elements in the web page. An external style sheet allows to keep all  CSS rules in one central location, which can then be linked to multiple HTML documents. This approach gives advanage of consistent styling across all the web pages in a web application. In future if any change needs to be incorporated, it can be done at one place.
Basic syntax of defining style rule :
selector {
      property: value;
      /* Additional properties and values */
}
Example :
/* Selects all <h1> elements and sets their color to red */
h1 {
    color: red;
}
Selector could be simple tag name or can use any combination of different types of selectors. All attributes are predefined and for few attibutes, some set of values are predefined 
e.g. border-style : solid/dashed/dotted etc.

Internal style sheet
An internal style sheet is a method of defining CSS rules directly within an HTML document using the <style> element. This allows you to specify the styling for the HTML elements contained within the same document. Internal style sheets are preferred when a unique styles needs to be applied to a single HTML document. The <style> element is used within the <head> section to contain the CSS rules. Inside the <style> element, CSS rules are written in the same way as in external style sheets
HTML
<html>
<head>
    <link href="external.css" rel="stylesheet" type="text/css" />
    <style>
        a{
             text-decoration: none;

        p{
            word-spacing: 8px;
        }
        ul{
           type: square

</head>

Inline style
Inline styles are a way of applying CSS directly to individual HTML elements using the style attribute. This method allows you to style specific elements without the need for separate CSS files or internal style sheets. Inside the style attribute, CSS rules are written directly as a string. Each rule is separated by a semicolon (;), and the property-value pairs are separated by a colon (:). 
Example: <p style="color:blue;background-color:gray" > .... </p>

Cascading Effect
One HTML page may be given styling by using different types of style sheets. This may create a conflict while applying styles to HTML pages.
If some properties have been defined for the one selector (element) in different style sheets, the value from the last read style sheet will be used.
External style sheet Internal style sheet
h1 {
  color: blue;
}
<style>
   h1 {
        color: orange;
   }
</style>
Please note :
  • If the internal style is defined after the link to the external style sheet, the <h1> elements will be "orange“.
  • If the internal style is defined before the link to the external style sheet, the <h1> elements will be "blue“
So the simple rule followed is the style sheet which is read by the browser at the end, will have its effect shown on the selected HTML element.

Cascading effect is really in the hands of the programmer because we should use different properties for the selected elements in differnt types of style sheet so that we will get combined effect of all the properties for the selected element.
