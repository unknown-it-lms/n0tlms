Notes: MySQL- Truncate clause

Truncate clause
The MySQL TRUNCATE TABLE statement is used to delete only the data of an existing table. Structure of he table remains intact.

This command helps to TRUNCATE a table completely with single command insted of deleting rows one by one.  
This is a DDL statement, hence records deleted using truncate will not be recovered by any command.
TRUNCATE is a DDL(Data Definition Language) command and is used to delete all the rows or tuples from a table. Truncate dose not support where clause, hence all rows will be deleted. So if the requirement is to delete all rows from the relation, then truncate can be used. Since it is DDL command, it cannot be rolled back.  TRUNCATE command is fast.  ​​​​​​​
Syntax:

TRUNCATE TABLE table_name

​​​​​​​Here all rows from table product will be deleted, but structure is preserved. Hence, describe structure will show the table structure, but data set will be empty.
 
Truncating table structure
Deletion of tables rows in DBMS is done using DELETE Clause or a TRUNCATE  Clause. Let us see the comparison between delete and truncate.   
Example :
TRUNCATE TABLE emp;
This command will delete all the data from emp table and this data will not be recovered.
