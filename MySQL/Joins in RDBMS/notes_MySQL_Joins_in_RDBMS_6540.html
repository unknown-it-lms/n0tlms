Concept   
Accessing Data From Multiple Tables    
Sometimes we need to use data from more than one table. Consider the example where, Employee IDs exist in the EMPLOYEES table. Department IDs exist in both the EMPLOYEES and DEPARTMENTS tables. Department names exist in the DEPARTMENTS table. In theis case, to produce the report, which contains employee_id, department_id and department_name, we need to derive data from 2 tables, employees and departments. Here, concept of JOINS is implemented. So, to retrive data from multiple tables, joins are implemented. Now when we consider 2 different tables , for data retrival, column in the table is accessed as table_name.column_name or alias is used.
Syntax:
    table1.column denotes the table and column from which data is retrieved.

 Cross Join    
Basic join in the concept of joins . The SQL CROSS JOIN produces a result set which is the number of rows in the first table multiplied by the number of rows in the second table if no 'WHERE clause' is used.  This kind of result is called as Cartesian Product.
If WHERE clause is used with CROSS JOIN, it functions like an INNER /NATURAL JOIN.
​
 SELECT e.ename, e.deptno,e.job , d.deptno                                    
 FROM emp e, dept  d;
 OR
 SELECT ename, deptno,sal
 FROM emp
 CROSS JOIN dept;

Above queries create a cartesian product of two tables. In the above diagram Table A contains 3 rows and table B contains 3 rows. Output generated after cartasian product will give 4*3 =12 rows.

Inner Join   
Tables can be joined automatically based on columns, in the  tables that have matching data types and names. Keyword NATURAL JOIN /INNER JOIN is used to achieve this.   
Note:: This join can happen only with those columns that have the same names and same data types in both tables. If the columns have the same name but different data types, then  the NATURAL JOIN/INNER JOIN syntax causes an error.
SELECT                 deptno, dname,location_id, city
FROM                   dept
NATURAL JOIN    locations;

In the above example, the LOCATIONS table is joined to the DEPT table by the LOCATION_ID column, which is the only column of the same name in both tables. If other common columns were present, the join would have used them all.
Natural Joins with a WHERE Clause 
Additional restrictions on a natural join can be implemented by using a WHERE clause. The following example limits the rows of output to those with a deptno equal to 20 or 50.
SELECT                deptno, dname, location_id, city
FROM                 dept
NATURAL JOIN /INNER JOIN  locations
WHERE                deptno IN (20, 50);

Note 
  •     JOIN table USING column_name performs an equijoin based on the column name.
  •     LEFT / RIGHT / FULL OUTER is used to perform outer joins.
  •     CROSS JOIN returns a Cartesian product from the two tables 

Equi Join
EQUI JOIN performs a JOIN against equality or matching column(s) values of the associated tables. An equal sign (=) is used as comparison operator in the where clause to refer equality.
EQUI JOIN may be performed by using JOIN keyword followed by ON keyword and then specifying names of the columns along with their associated tables to check equality.
SELECT column_list
FROM table1, table2....
WHERE table1.column_name = table2.column_name;    
OR
SELECT *
FROM table1
JOIN/EQUI JOIN table2
[ON (join_condition)];

Example::
SELECT eno,ename,sal,e.depto,d.dname
FROM emp e, dept d
WHERE e.deptno =d.deptno;
OR
SELECT eno,ename,sal,e.depto,d.dname
FROM emp e,
JOIN  dept d
ON  e.deptno =d.deptno;    

Non-Equi Join
A nonequijoin is a join condition containing  operator other than  equality operator. The relationship between the emp table and the salgrade table is an example of a nonequijoin. A relationship between the two tables is that the sal  column in the emp table must be between the values in the losal and hisal columns of the salgrade table. The relationship is obtained using BETWEEN ..AND i.e operator other than equality (=).
SELECT        e.ename, e.sal, s.grade
FROM         emp  e
JOIN            salgrade s
ON               e.sal
BETWEEN   s.losal   AND   s.hisal;
Example creates a nonequijoin to evaluate salary grade of an employee. The salary must be between any pair of the low and high salary ranges. It is important to note that all employees appear exactly once when this query is executed. No employee is repeated in the list.
Note 
Other conditions (such as <= and >=) can be used, but BETWEEN is the simplest. Remember to specify the low value first and the high value last for BETWEEN clause.
An nonequi join is an inner join statement that uses an unequal operation (i.e.: <>, >, <,!=, BETWEEN, etc.) to match rows from different tables.

Example 
SELECT  e.empno,e.ename,e.job,d.dep_name
FROM   emp e,dep  d
WHERE  e.deptno > d.deptno;

Self Join
​​​​​​​Sometimes we need to join a table to itself. To find the name of the manager of each employees. This is because the output which we want to generate  takes the data from only one table. We need to join the emp table to itself i.e perform a self join.
For example, to find the name of Martin's manager, we need to:
          --Find Martin in the emp table by looking at the ename column.
           --Find the manager number for Martin by looking at the mgr column.
In this process, we look in the table twice. The first time you look in the table to find Martin in the ename column and mgr value. The second time you look in the eno column to find mgr value  and the then ename  column to find mgr name.
The ON clause can also be used to join columns that have different names, within the same table or in a different table.
The example shown is a self-join of the e

mp table, based on the eno and mgr columns.
SELECT    e.eno, e.ename, e.depno, d.deptno, d.location_id
FROM     emp  e
JOIN        dept  d
ON          e.deptno = d.deptno    AND        e.manager_id = 149;

Left Outer Join
An outer join is similar to equijoin but it gets also the non-matched rows from the table. It is categorized in Left Outer Join, Right Outer Join and Full Outer Join .
Left Outer Join returns all rows from the left (first) table specified in the ON condition and only those rowfrom the right (second) table where the join condition is met.

​

Syntax 
SELECT columns  
FROM table1  
LEFT JOIN table2  
ON table1.column = table2.column;  
This left outer join displays all matching records of both table along with the records in left hand side table of join clause which are not in right hand side table of join clause.

Example 
SELECT eno,ename,job ,dname
FROM  emp e
LEFT OUTER JOIN dept d
ON  e.deptno=d.deptno;
OR
SELECT    eno,ename,job,dname
FROM     emp  e, dept d
WHERE    e.deptno=d.deptno(+);

Right Outer Join 
The Right Outer Join returns all rows from the right-hand table specified in the ON condition and only those rows from the other table where the join condition is satisfied.

Syntax 

SELECT columns  
FROM table1  
RIGHT [OUTER] JOIN table2  
ON table1.column = table2.column;  
​
 
Example 
SELECT   eno,ename,job ,dname,location    
FROM    emp e RIGHT OUTER JOIN dept d
ON      e.dept_no=d.dept_no;
OR
SELECT   eno,ename,job ,dname,location
FROM    emp  e, dept d
WHERE    e.deptno(+)=d.deptno;

This right outer join displays all matching records of both tables along with the records in left hand side table of join clause which are not in right hand side table of join clause.

 Full Outer Join
The Full Outer Join returns all rows from the left hand table and right hand table. It places NULL where the join condition is not met.

​
Syntax::
SELECT columns  
FROM table1  
FULL [OUTER] JOIN table2     
ON table1.column = table2.column; 
A full outer join returns all rows from both the tables. Left and right side of the join clause will be extended with nulls if they do not satisfy the join condition.
MySQL dose not support FULL JOIN. This works for ORACLE. So in MySQL to get the result as like FULL JOIN ,
Union operator is used. This query will be the combination of Left outer join and Right outer join.
