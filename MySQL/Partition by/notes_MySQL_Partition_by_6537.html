Hence, a user is restricted in selecting any column except the column used in group by clause.  Partition by clause is one step ahead. This clause will allow user to display any number of aggregate functions along with the columns in a select clause.
 
Partition by clause will partition the data (will create the partitions) i.e group the data same as in group by clause, but without any limitations of group by clause.   
      
When partitions are created , different portions of a table are stored as separate tables in different locations. The condition on which division of data is accomplished is known as a partitioning function.  The function is selected according to the partitioning type specified by the user, and takes the value of a user-supplied expression as a prameter. This expression can be a column value, a function acting on one or more column values, or a set of one or more column values, depending on the type of partitioning that is used.

Implementation
Data is grouped(partitioned) to perform certain operations which include aggregate functions. These created partitions are called as windows.  Functions will work on each window created. Hence these functions are also called as window functions.   
over() function
To partition the data we need over() function. This function takes partition by clause as a parameter and works accordingly. Scope of all  functions used after partitioning the data is window created by that partition. If data is not partitioned, the scope of Over() function without parameter is entire table.
