Handling multiple tables
Many a times, the data is scattered in multiple tables and need to be accessed. In this case we need to access multiple tables. There are many ways to access multiple tables, like using condition on common column in tables, joins and set operators.
Multi table access   
Using Where Clause
Let us consider two tables new_emp and dept .
Below is the data populated in the tables.

Now, the requirement is to display empno,ename,sal,deptno and dname for all employees.
dname is in dept table and all the rest information is in new_emp table. Now in such scenario the data in the tables is accessed using common column. In this case the column is dptno. This column shares same information(data) among two tables. Column name may differ in the tables. 
Following query will generate empno,ename,sal,deptno and dname for all employees. 
SELECT e.emmpno, e.ename, e.sal, e.deptno, d.dname
FROM emp e, dept d
WHERE e.deptno = d.deptno;

Set operators
 1. Union   
The UNION  operator is used to combine the result-set of two or more SELECT statements.
There are certain things which we need to follow while using UNION operator, they are as follows 
  • Every SELECT statement within UNION must have the same number of columns.
  • The columns must also have similar data types.
  • The columns in every SELECT statement must also be in the same order.

3. Intersect
Intersect operator returns rows that are common to both the result set.
Table t1Table t2
​​
Query to combine data sets in t1 and t2 is as follows.
select * from t1
intersect
select * from t2;   
​
Intersect generates result set only containing common rows.
4. Except  (minus)
The EXCEPT operator is used to deduct the results of one query from the results of another. In other words, it only returns the entries unique to the first query and which do not appear in the second query.
Table t1Table t2
​​
select * from t1
except
select * from t2;   
Above query will all rows from table t1 which do not exist in table t2.
