In-built functions in MySQL
MyQL has a rich library of in-built functions which can be used to perform various operations. The functions can be used for case conversion of strings, in-string or substring operations, mathematical computations on numeric data, and date operations on date type values.                                      
On a broader category, there are two types of functions :-
1. Single Row functions (Scalar functions)
   Single row functions are the one who work on single row and  return one output per row.
   For example,
   length and case conversion functions are single row functions.
2. Multiple Row functions (Aggregate functions)    
   Multiple row functions work upon group of rows and return one result for the complete set of rows.
   They are also known as Group Functions.
    For example,
    max,min,avg functions are group functins.

Let us study this in details.

Using Aggregate Functions for restricting records
To restrict records using aggregate functions , we need to use having caluse.  Any time whenever a condition is to be checked on the result generated by aggregate functions, where caluse is not applicable, we need to use having clause. Where caluse is used to check the condition imposed on the column of the table. Having clause is used to check the condition imposed by aggregate functions.
Aggregate functions are group functions, which require GROUP BY clause to group records according to the columns. After using GROUP BY to filter records with aggregate functions like COUNT,  HAVING clause can be used. HAVING clause can be  used only when  group by clause comes into picture.

In the example, we compare whether COUNT( ) is more than 2.

SELECT category, COUNT(*)
FROM product
GROUP BY category
HAVING COUNT(*)>2;

Here condition is on aggregate function count(), hence having clause is used.
Query will fetch caregory from product table if there are more than 2 products belonging to the same category.
