Notes: MySQL- Views in RDBMS

Views in RDBMS
Views in MySQL are  “virtual tables” that are used to view data from one or more tables. Views do not have their data but rather store data virtually, consisting of rows and columns. Views are very helpful in restricting access to your application’s critical data to third-party users. Views in MySQL can be created by selecting some/all columns and some/all rows of a table by filtering out the rows based on some condition(s). 
Views are stored queries that when invoked produce a result set. A view acts as a virtual table. Views  are only a structure, and contain no data. Their purpose is to allow a user to see a subset of the actual data.  
A view contains rows and columns, just like a real table. The fields in a view are fields from one or more real tables in the database.
SQL statements and functions can be added  to a view and present the data as if the data comes  from one single table.



Need of views
Simplicity
Instead of writing complex joins & queries, views provide a way of writing SELECT statements which is used to fetch required data from multiple tables. This data than can be handled using a view, which is more simpler form than he complex query.

Security 
Views expose only the data to the third-party apps and hide the internal details like table structure, table name, attributes, etc, thus adding extra security.

Consistency
By writing views instead of common queries, we can write a view that avoids multiple declarations & definitions of the same queries and eventually provides a centralized way. Just like the normal tables, you can perform operations like create, update, drop, etc., on the views.

Creating View from multiple tables
In this example we will create a View named EmpDetails from two tables emp and dept.
CREATE  VIEW  EmpDetails  AS
SELECT   emp.eno, emp.ename, emp.deptno, dept.dname
FROM     emp, dept
WHERE   emp.deptno  = dept.deptno;
Here a view is created by selecting 2 different tables emp and dept.

Deleting Views   
Views can also be deleted if not required. DROP command is used to delete the view permanently.
Syntax:
DROP  VIEW view_name;
Example..
DROP  VIEW student_view;
This command will delete view by name student_view permanently.
Updating Views 
We can make changes in the views by updating it. For updating the data in view , certain conditions needed to be satisfied .
Rules for updation
  • The SELECT statement which is used to create the view should not include GROUP BY clause or ORDER BY clause.
  • The SELECT statement should not have the DISTINCT keyword.
  • The view should have all NOT NULL values.
  • The view should not be created using nested queries or complex queries.
  • If the view is created using multiple tables then we will not be allowed to update the view.
Altering Views (Adding/Removing columns from view )
1. Using CREATE OR REPLACE
CREATE OR REPLACE VIEW statement is used to add/remove columns from view .
Syntax ::
CREATE  OR  REPLACE  VIEW  view_name  AS
SELECT column1,coulmn2,..
FROM table_name
WHERE condition;

Example...
CREATE  OR  REPLACE  VIEW  MarksView  
AS
SELECT  StudentDetails.NAME,  StudentDetails.ADDRESS, StudentMarks.MARKS,  StudentMarks.AGE
FROM StudentDetails,  StudentMarks
WHERE StudentDetails.NAME  =  StudentMarks.NAME;

When using create or replace view following things cannot be done
   1. cannot change column name using as.
   2. cannot change the datatype of column.
   3. cannot change order of column, if needed new colum must be added at the end in select column list.

2. Using Alter clause
  • Changing column name of the view 
    Syntax
ALTER VIEW view_name RENAME COLUMN old_column_name TO new_column_name ;
e.g
ALTER VIEW dept10 RENAME COLUMN deptno TO dno;
Here,  
column name deptno is changed to dno.
  • Changing name of the View
Syntax
ALTER VIEW dept10 RENAME TO new_dept10;
Here,  
view by name dept10  is changed to new_dept10.
3. Inserting Row in a View 
When a modifiable view contains no derived columns, row can be inserted into it as if it were a table. The database server, however, uses NULL as the value for any column that is not exposed by the view. If such a column does not allow NULL values, an error occurs, and the insert fails.
Syntax ::
INSERT  INTO view_name(column1, column2 , column3,..)
VALUES(value1, value2, value3..);
view_name  :  Name of the View
Example.......
INSERT INTO  DetailsView(NAME, ADDRESS)
VALUES("Ajay","Pune");
This will insert a row in the view.
4. Deleting Row in a View 
We use the DELETE statement of SQL to delete rows from a view.
Deleting a row from a view first deletes the row from the actual table and the change is then reflected in the view.
Syntax ::
DELETE FROM view_name
WHERE condition;
view_name : Name of view from where we want to delete rows
condition     : Condition to select rows
Example :
DELETE
FROM Emp_view
Where ename=’Smith’;
Here record containing name as ‘Smith’ will be deleted from base table as well as a view.
With  CHECK OPTION 
This clause is used for an updatable view . This  prevents  the invisible rows from being updated .
The WITH CHECK OPTION clause is an integrity constraint option that restricts the rows in the table that can be affected by an INSERT or UPDATE request to those defined by the WHERE clause.
If WITH CHECK OPTION is not given, then the integrity constraints specified by the WHERE clause are not checked for updates.

Handling Views
Since views are virtual table, all operations which are done on any table can be performed on views as well. Let us study these operations on views..
Creating Views

1. From a single table (Simple View)
When a view is created on a single table, it is called as a simple view.
In this example we will create a View named DetailsView from the table StudentDetails.
Syntax
CREATE VIEW view_name AS
     SELECT column_list
     FROM table_name
     [  WHERE condition  ] ;

view_name:   Name for the View
table_name:  Name of the table
condition    :   Condition to select rows
e.g
CREATE VIEW   DetailsView   AS
SELECT      NAME, ADDRESS   
FROM       StudentDetails
WHERE     S_ID < 5;
