Notes: MySQL- Grouping Data

Grouping Data
Some times we need to process the data in a group. Creating this kind of group is a logical requirement . In such scenarios we use group by clause to create logical groups, and then we process this data. Mysql library gives us certain functions to process this grouped data. These functions are called as group functions or aggregate functions.  These aggregate functions will work on all the created groups.
Group by
 Some most common group functions.
Function Description

COUNTNumber of rows .

SUMSum values of n, ignoring null values.

General Syntax for Group BY Functions::
   SELECT            [column,] group_function(column),....
   FROM              table
   [WHERE          condition]
   [GROUP BY     column]
   [HAVING          condition]
   [ORDER BY      column];


Example::
1. MAX Function
    SELECT  max(salary)
    FROM   employees;
 Will fetch the maximun salary among all the employee.
2. COUNT Function
    The COUNT function has three formats:
    1. COUNT(*)
       Returns the number of rows in a table that satisfy the criteria of the SELECT statement, including duplicate rows and rows containing null values in any of the columns.
    2. COUNT(expr)
        Returns the number of non-null values that are in the column identified by exp.
    3. COUNT(DISTINCT expr)
        Returns the number of unique, non-null values that are in the column identified by expr.
Group Functions and Null Values
  •     All group functions ignore null values in the column.
   
DISTINCT Keyword
    Use the DISTINCT keyword to suppress the counting of any duplicate values in a column.     

GROUP BY Clause
GROUP BY clause is used  to divide the table rows into groups. Group functions return summary information for each group.
    e.g
    SELECT         AVG(salary)
    FROM          employees
    GROUP BY   department_id;
 
    You can use the group function in the ORDER BY clause:
    SELECT         department_id, AVG(salary)
    FROM           employees
    GROUP BY   department_id
    ORDER BY    AVG(salary);
Using Aggregate Functions for restricting records
To restrict records using aggregate functions , we need to use having caluse.  Any time whenever a condition is to be checked on the result generated by aggregate functions, where caluse is not applicable, we need to use having clause. Where caluse is used to check the condition imposed on the column of the table. Having clause is used to check the condition imposed by aggregate functions.
Aggregate functions are group functions, which require GROUP BY clause to group records according to columns. After using GROUP BY to filter records with aggregate functions like COUNT,  HAVING clause can be used. HAVING clause can be  used only when  group by clause comes into the picture.

In the example, we compare whether COUNT(*) 

SELECT category, COUNT(*)
FROM product
GROUP BY category
HAVING COUNT(*)>2;    

having clause
HAVING clause is used to specify which groups are to be displayed, thus further restricting the groups on the basis of aggregate information.
MySQL server performs the following steps when HAVING clause is used.
  • Rows are grouped.
  • The group function is applied to the group.
  • The groups that match the criteria in the HAVING clause are displayed.
Using HAVING Clause  
  •      GROUP BY clause can be used without using a group function in the SELECT list.
  •      If we restrict rows based on the result of a group function, we must have a GROUP BY clause as well as the       HAVING clause.
The following example displays the department numbers and average salaries for those  departments with a maximum salary that is greater than $10,000.
    e.g
    SELECT         department_id, AVG(salary)
    FROM           employees
    GROUP BY   department_id
    HAVING       max(salary)>10000; 
Note ::
  Having clause is used, when the condition is checked on Aggregate functions.
