Notes: MySQL- Window Function

Window Function
Window Functions in MySQL/Oracle work like aggregate functions, with a very  little difference. Aggregate functions group  the query rows into a single row to generate a  result. When we apply window functions, MySQL/Oracle will create loigical windows and the functions will be applied on created windows.  These functions return a result for each created window. Query rows are rows on which function will be applied . Means window operations do not collapse group of query rows into a single row. These functions help in performing data analysis, calculations, making decisions  which gives it an advantage over aggregate functions.

It was in 2018, that MySQL introduced a new feature  'window functions' , which is accessed via the OVER clause. Window functions are a super powerful resource now available in almost all SQL databases. They perform a specific calculation (e.g. sum, count, average, etc.) on a set of rows and this set of rows is called a “window” and is defined by the MySQL using OVER clause.
Window functions have been available in most major databases for quite some time, but until 2018 they were not available in MySQL. To keep MySQL up to date, window functions were introduced in MySQL 8.02.

Working of window functions
A window function performs an aggregate-like operation on a set of query rows.  An aggregate operation groups query rows into a single result row, where as window function produces a result for each query row.

The row for which function evaluation occurs is called the current row.
The query rows related to the current row over which function evaluation occurs comprise the window for the current row.

For example, using the sales information table, these two queries perform aggregate operations that produce a single global sum for all rows taken as a group, and sums grouped per country:

On the other hand,  window operations do not collapse groups of query rows to a single output row. Instead, they produce a result for each row. Like above queries, the following query uses SUM(), but this time, sum() is a window function.

​

Each window operation in the query is signified by inclusion of an OVER clause that specifies how to partition query rows into groups for processing by the window function:

  • The first OVER clause is empty, which treats the entire set of query rows as a single partition. The window function thus produces a global sum, but does so for each row.

  • The second OVER clause partitions rows by country, producing a sum per partition (per country). The function produces this sum for each partition row.

Window functions are permitted only in the select list and ORDER BY clause. Query result rows are determined from the FROM clause, after WHERE, GROUP BY, and HAVING processing, and windowing execution occurs before ORDER BY, LIMIT,  and  SELECT DISTINCT.

The OVER clause is permitted for many aggregate functions, which therefore can be used as window or nonwindow functions, depending on whether the OVER clause is present or absent.

RANK ( ) function
The rank ( ) function is also a type of ranking function also used as an analytical function. It calculates the rank of a value in a set of values. The rank is same for the same values. In this example, we are going to use the RANK ( ) function to calculate the rank based on the profit. For this we first partitioned the rows on the basis of the country so each country is one unique partition and then we use order by clause to sort it into descending order based on the profit column and then apply the rank () window function on that. Let us look at the query below
SELECT empno,ename,deptno,sal ,rank() over(partition by deptno order by sal)rnk

DENSE_RANK ( ) function
The DENSE_RANK ( ) function is also a type of ranking function which calculates the rank of a value in a set of value and same values gets the same rank. The difference between rank () and dense_rank () is that in case of dense_rank () function, we get rank values as consecutive integers meaning that the next consecutive rank is not skipped in case of ties. Let us consider the previous example with dense_rank () function instead of rank () function.

Cumulative sum
​​​​​​​A cumulative sum is a running total of values progressively increasing or accumulating over time or across a dataset. It represents the summation of all values encountered up to a certain point or row in the dataset, where each new value adds to the total of the previous values.  

In SQL, the "ROWS BETWEEN" clause is a powerful feature that allows you to define a window of rows for analysis or calculation within a query. By specifying a range of rows, you can perform calculations or apply aggregate functions to a subset of data rather than the entire result set.  

Example:
select  date, amount, sum(amount) over(order by date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) as cumulativesum
from table;   

The "ROWS BETWEEN" clause is  used in conjunction with window functions, which are special functions in SQL that operate on a specific subset of rows known as a window. The "ROWS BETWEEN" clause enables you to define the boundaries of the window, specifying which rows should be included in the calculation. 
Rows Between Clause
The syntax is:

ROWS BETWEEN lower_bound AND upper_bound

The bounds can be any of these five options 
  1. UNBOUNDED PRECEDING – All rows before the current row.
  2. n PRECEDING – n rows before the current row.
  3. CURRENT ROW – Just the current row.
  4. n FOLLOWING – n rows after the current row.
  5. UNBOUNDED FOLLOWING – All rows after the current row.



OVER (PARTITION BY 'field' ORDER BY 'field' ROW 'field') 

Over: clause responsible for calling the function.
Partition by: divides the partition rows based on the defined field
Order by: order rows within each partitioned field
Rows: specifies the window frame relative to the current row

In addition, 'Rows' can follow the following clauses:

Unbounded Preceding: all rows before current row.
Current Row: current row only.
Unbounded Following: all rows after the current row.
N Preceding: 'n' rows before the current row.
N Following: 'n' rows after the current row.

To create Window Functions, it is not mandatory for a  function to contain all those clauses,.
How to Create a Cumulative Sum 
Creating this type of sum is very simple and for it development we will use ORDER BY and ROWS clauses. Furthermore, in the Over clause we will need to specify ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW.

select hiredate,sal,sum(sal) over( ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) as cumulativesum
from emp;  

Same as  

select hiredate,sal,sum(sal) over( ROWS UNBOUNDED PRECEDING   ) as cumulativesum
from emp;  

The sum of daily_sales first appears as a simple aggregate function. However, the keyword OVER means this is an ordered analytic, not an aggregate. When you see the keyword OVER, the system will have an ORDER BY statement in parenthesis that sorts the rows before the analytic calculation. Once the rows are ordered by sale_date, each new daily_sales value will be added to the total of the previous values.

We also have a WHERE clause, which only places rows with a product_id value of 1000 or 2000 in the data set to be ordered and calculated.

select deptno,sal,sum(sal) over( ROWS  UNBOUNDED PRECEDING   ) as cumulativesum
from emp
where deptno in (10,20);

select deptno,sal,sum(sal) over(partition by deptno ROWS UNBOUNDED PRECEDING ) as subtot, sum(sal) over( ROWS UNBOUNDED PRECEDING ) as grandtot
from emp;
