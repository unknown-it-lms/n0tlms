Restricting records(filtering records)
Sometimes operations need to be preformed on certain set of rows. This can be achieved using where clause as well as regular expressions in any RDBMS.
Specific rows can be selected by adding a WHERE clause to a SELECT query or by using operator REGEXP . WHERE clause appears just after the FROM clause in SELECT query hierarchy and REGEXP operator is used along with where clause. The sequence of all the clauses used in the query has to be maintained in all scenarios. If violated,  MySQL will raise an exception and an error will be generated.
​​​​​​​

where clause
Limiting the selected rows 
Syntax:
SELECT *
FROM table
[WHERE condition(s)]       
    
In the syntax,
  • WHERE  is the keyword
  • [condition] contains column names, expressions, constants, literals and a comparison operator.

Points to be noted 
  • A SELECT clause can contain only one WHERE clause. However, multiple filter conditions can be appended to WHERE clause using AND or OR operator.
  • The columns, literals or expressions in a predicate clause must be of similar or interconvertibl data types.
  • Column alias(discussed later) cannot be used in the WHERE clause.
  • Character literals must be enclosed within single quotation marks and are case sensitive.
  • Date literals must be enclosed within single quotation marks and are format sensitive.

For checking the multiple conditions on multiple columns use of operators can be done while writing a query.
    
Using  Operators
1. Comparison Operators
Comparison operators are used  to compare one term or operand with another term. SQL offers comprehensive set of equality, inequality and miscellaneous operators. They can be used depending on the data and filter condition logic in the SELECT query. When you use comparison operators in a WHERE clause, the arguments (objects or values you are comparing) on both sides of the operator must be either a column name, or a specific value. If a specific value is used, then the value must be either a numeric value or a literal string. If the value is a character string or date, you must enter the value within single quotation  marks (' ').

2. Logical Operators  
Multiple filter conditions can be added to the WHERE clause predicate. More than one condition can be combined together using logical operators AND, OR and NOT.
AND: joins two or more conditions, and returns results only when all of the conditions are true.
         The AND operator links two or more conditions in a WHERE clause and returns TRUE only if all the
         conditions are true.
         e.g
         SELECT        last_name , first_name , Gender
         FROM         employees
         WHERE        Gender = 'F'   AND   salary > 20000;

OR : joins two or more conditions, and it returns results when any of the conditions are true.
The OR operator links more than one condition in a WHERE clause and returns TRUE if either of the
         condition returns true.
         e.g
         SELECT        last_name , first_name , Gender,depertment_id
         FROM         employees
         WHERE        department_id=100  OR  department_id=200

NOT: negates the expression that follows it.
         The NOT operator is used to negate an expression or conition.
         e.g
         SELECT        last_name , first_name , Gender,depertment_id
         FROM         employees
         WHERE       department_id  IS NOT NULL;
​​​​​​​In the above query all employees will be displayed whose department_id is not null.
3. BETWEEN..AND Operator
The BETWEEN operator can be used to compare a column value within a definite range. The specified range must have a lower and upper limit where both are inclusive during comparison. Its use is similar to composite inequality operator (<= and >=). It can be used with numeric, character and date type values.
Syntax
SELECT column list
FROM  table_name
WHERE column_name  BETWEEN small_value and bigger_value;
e.g
SELECT  last_name,salary
FROM   employees
WHERE salary BETWEEN 1500 and 2500;

4. IN Operator
The IN operator is used to test a column value in a given set of value. If the column can be equated to any of the values from the given set, the condition is validated. The condition defined using the IN operator is also known as the membership condition.
e.g
SELECT   id,last_name,salary,department_id
FROM    employees
WHERE   department_id  IN (100,102,200);

This will list the information of employees from department no 100,102 and 200

5. Like Operator
The LIKE operator is used for pattern matching and wildcard searches in a SELECT query. If a portion of the column value is unknown, wildcard can be used to substitute the unknown part. It uses wildcard operators to build up the search string, thus search is known as Wildcard search.
Two operators used for search are Percentile ('%') and Underscore ('_'). Underscore ('_') substitutes a single character while percentile ('%') represents more than one characters. They can be used in combination as well.
For example, the below SELECT query lists the first names of those employees whose last name starts with SA.
SELECT   first_name
FROM    employees     
WHERE   last_name  LIKE  'SA%';
6. Is operator   
IS NULL Condition  / IS (NOT) NULL Condition  
NULL values cannot be tested using equality operator. It is because NULL values are unknown and unassigned while equality operator tests for a definite value. The IS NULL operator serves as equality operator to check NULL values of a column.
e.g
SELECT   id,last_name,salary,department_id
FROM    employees
WHERE   department_id  IS NULL;
Here department_id is checked for null and if the value in the column is null , then the query will fetch the rows whose department_is is null. Here, since null is not a numeric  or a string value, null will be checked using IS operator. We cannot use '=' operator.
e.g
SELECT   id,last_name,salary,department_id
FROM    employees
WHERE   department_id  IS NULL;
 
Regular expressions   
It provides a powerful and flexible pattern match that can help us implement power search utilities for our database systems. REGEXP is an operator used while performing  pattern matches. RLIKE is the synonym. It also supports a number of metacharacters which allow more flexibility and control while performing pattern matching. 
PatternMatch
*Zero or more instances of string preceding it.
+One or more instances of strings preceding it.
.Any single character
?Match zero or one instances of the strings preceding it.
^caret(^) matches Beginning of string
$End of string
[abc]Any character listed between the square brackets
[^abc]Any character not listed between the square brackets
[A-Z]Match any upper case letter.
[a-z]Match any lower case letter.
[0-9]Match any digit from 0 through to 9.
p1|p2|p3Matches any of the patterns p1, p2, or p3
{n}n instances of preceding element
{m,n}m through n instances of preceding element 

Example  

-Display all names starting with letter A.
select *
from emp
where REGEXP_LIKE(ename,'^A');

-Name ending with letter R
select *
from emp
where REGEXP_LIKE(ename,'N$');

-Pattern where N can come any where
select *
from emp  
where REGEXP_LIKE(ename,'.*N.*');

-start either A or M or S
select *   
from emp
where REGEXP_LIKE(ename,'^[AMS]');

-starts with M, L at third position and ends with R
select *
from emp
where REGEXP_LIKE(ename,'^M.L.*R$');

-Name starts with A and ends with N or starts with M and ends with R or starts with S and ends with H. 
select *
from emp
where REGEXP_LIKE(ename,'^A.*NS | ^M.*R$ | ^S.*H$' );

-Do not start with A
select *
from emp
where REGEXP_LIKE(ename,'^[^A]');
