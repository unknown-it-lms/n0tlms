Notes: MySQL- Subqueries

Subqueries
A MySQL nested query, also called as sub query, is a query nested within another query such as SELECT, INSERT,UPDATE or DELETE. Also, a subquery can be nested within another subquery.  A  subquery is called an inner query while the query that contains the subquery is called an outer query. A subquery can be used anywhere and must be closed in parenthesis.     
Nested Queries
Using select clause in nested queries
While using select clause in subquery, inner query will be executed first and then the outer query. Both inner and outer query are independent of each other and can execute seperately.
Inner query may fetch  a single row or multiple rows. Depending on the number of rows fetched by inner query, a comparison operatore will be used. 
Syntax 
SELECT column_list
FROM table_name
WHERE condition comparison_operator(SELECT  column_name
                                                                         FROM table_name
                                                                        [ WHERE condition ]);
1. Single-row subquery
Consider an example in which ,we want to display the list of all employees who work in the department where Smith works.
e.g  
select ename,deptno         
​​​​​​​from   emp 
where deptno = (select deptno
                               from emp
                               where ename='Smith' );
In this case inner query(child query) executes first. So deptno of Smith is fetched and is compared with the deptno of all other rows in the table. Rows which match the deptno of Smith will  be finally fetched by the query. In the above example inner query is fetching only 1 row(values). Henc it is called as single row subquery.
Following operators are used to handle single row subquery.

2. Multiple-row subquery 
Queries that return more than one row from the inner SELECT statement are called as multirow subqueries.
These queries explicitly need special operators called as Muli-Row operators to generate  proper output.
Operator
Meaning
IN
Equal to any member in the list .
ANY
Compare value to each value returned by the subquery .
ALL
Compare value to every value returned by the subquery .
Example
Find the employees who earn the same salary as the minimum salary for each department.
The inner query is executed first, producing a query result. The main query block is then processed and it uses the values that were returned by the inner query to complete its search condition. Here there can be multiple
employees satisfying the above condition, so a subquery may return multiple records. Hence  '=' operator will not work.
So the query will go as follows....

Using IN
SELECT   last_name, salary, department_id
FROM     employees
WHERE   salary IN  (SELECT        MIN(salary)
                                   FROM          employees
                                   BY                department_id);
Here outer query will search in the set of records returned by inner query and finally query will fetch all the records which satisfy the condition.

Using ANY
 Display employees who earn salary less than Smith or Ward.
select  *
from emp
where sal > any (select sal
                             from emp   
                             where ename in('Smith','Ward'));
In above situation, inner query returns more than one row, hence any is used for comparison. Query will return a row which satisfies at least 1 condition.

Using ALL
Display employees who earn salary more than Smith and Ward.
select  *
from emp
where sal > all (select sal
                            from emp
                            where ename in('Smith','Ward'));
In above situation, inner query returns more than one row, hence all is used for comparison. Here all will compare all the values in the set and will return the row which satisfies both the conditions.  ALL means that the condition will be true only if the operation is true for all values in the range.
  
Note 
--ALL means more than the maximum, and <ALL means less than the minimum. The NOT operator can  be used with IN, ANY, and ALL operators.
--There are also multiple-column subqueries, which are queries that return more than one column  from the inner SELECT statement.  
Insert,update, delete in subquery   
Sometimes data to be deleted or updated  from the table is not directly available. In this case we need to fetch initial information using subquery and then delete or update the data. Delete and update is a simultaneous read-write operation. MySQL engine cannot handle simultaneous read-write operations. It generates error stating 'You can't specify target table'. Hence we need to write delete and update subquery in a different format.
delete subquery
Sometimes data to be deleted from the table is not directly available. In this case we need to fetch initial information using subquery and then delete the data. Consider following example,
Delete all the employees working in Smith's department.
In this case we need to find department of Smith and then department of other employees. This query has 2 operations i.e modify(delete) and read(select) which work simultaneously. Considering standard approach, the query goes as follows,
delete
from emp11
where deptno = (select deptno
                              from emp11
                              where ename='Smith');
Now, above query generates an error "ERROR 1093 (HY000): You can't specify target table 'emp11' for update in FROM clause".  MySql, prevents this type of simultaneous access of read and write, and generates an error. To overcome this, we need a dummy table . Hence the query goes as follows......
delete
from emp11
where deptno=(select deptno
                            from (select deptno
                                       from emp11
                                       where ename='Smith')
                            tempTable);
Above mentioned tempTable will be created by the system. It is  a temporary table and will be deleted as soon as a query gets executed. We can give any tablename.

Here as well read/modify operatios take place simeltaneously generating the same error as in case of deletion of a row. Hence ,here again temporary table is required. Query will go as follows...

Correlated Query
When the inner query depends on the outer query for execution then it is called as correlated query.
Non-correlated subquery can be executed independently, but a correlated subquery is evaluated for each row processed by the outer query, making it potentially more resource-intensive but also useful for complex data retrieval scenarios.
 Key characteristics of correlated subqueries 
  • Dependency on Outer Query 
    Subquery reference columns or values from the outer query. 
  • Row-by-Row Execution 
    They are re-executed for each row processed by the outer query. 
  • Performance Impact
    Due to repeated execution, correlated subquery can lead to performance issues if not optimized, especially          with large datasets. 
  • Complexity 
    They are  more complex to write and understand compared to simple subqueries.
