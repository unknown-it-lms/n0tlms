Notes: MySQL- Indexing data in RDBMS

Concept of indexes
A database index is a data structure implemented to improve the speed of operations in a table. Indexes can be created using one or more columns, providing the basis for both rapid random lookups and efficient ordering of access to records. Index is used to retrieve data from the database more quickly . The users cannot see the indexes, they are just used to speed up searches/queries.
Updating a table with indexes takes more time than updating a table without (because the indexes also need an update). Hence , indexes are created on columns that are  frequently searched. Practically, indexes are also a type of tables, which keep primary key or index field and a pointer to each record into the actual table. The users cannot see the indexes, they are just used to speed up queries and will be used by the Database Search Engine to locate records very fast. Indexes on primary key ,unique constraint are automatically created.
The CREATE INDEX statement is used to create indexes in tables.

Advantages of indexes
1. Faster search
2. It also helps to retrieve data from related table i.e from other table
Disadvantages of indexes
1. Extra space is needed
2. It reduces speed of execution of DML (insert,delete, update)       
   ​​​​Types of indexes
Indexes in mysql are primarily of two types.
1. Clustered index
    clustered index is only one, it doesnot require extra space, and table data is arranged in  sorted order
    on field of clustered index.
2. Non clusterd index
    non clustered indexes can be  many.  We can generate it.  It needs extra space, all indexes are arranged in sorted oreder either ascending or descending order.
Apart from above mentioned indexes following are some other indexes.
1. fulltext index
     applied on keywords, and not on entire string.
2. descending index
      it stores data in descending order.
3. mulicolumn or composit index
      these indexes are created on multiple column
