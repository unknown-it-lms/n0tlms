Notes: MySQL- Select clause in details

Select clause in details
 Syntax :

SELECT column_list
FROM table_name_list
[ WHERE condition
GROUP BY column_name
HAVING condition
ORDER BY column_name
LIMIT int_val ; ]

Select clause is used to fetch the required data from any table. In the above syntax SELECT and FROM statements are mandatory, of all other statements. We can apply multiple condition, group the data, restrict rows or columns using above syntax. We can as well use arithmetic operators and case statements in select clause.
Alias and arithmatics in select clause
Selecting Records(Rows)/Fields(Columns)

SQL SELECT statement is used to  retrieve data from a table in the database. A query may retrieve information from specified columns or from all of the columns in the table. To write a simple SQL SELECT statement, we need to specify the column(s) name and the table name.
The whole query is called SQL SELECT Statement.

Syntax of SQL SELECT Statement:  
SELECT column_list
FROM table-name
[WHERE Clause]
[GROUP BY clause]   
[HAVING clause]
[ORDER BY clause];   

In the above syntax,
  • table-name is the name of the table from which the information is retrieved.
  • column_list includes one or more columns from which data is retrieved.
  • The code within the brackets is optional.

Operator Precedence    
  •     If an arithmetic expression contains more than one operator, multiplication and division are                                  evaluated first.
  •     If operators in an expression are of the same priority, then evaluation is done  from left to right.
  •     We can use parentheses to force the expression that is enclosed by parentheses to be evaluated first.
Rules of Precedence
  •       Multiplication and division occur before addition and subtraction.
  •       Operators of the same priority are evaluated from left to right.
  •       Parentheses are used to override the default precedence or to clarify the statemen

Defining a Column Alias
When displaying the result of a query, SQL Developer normally uses the name of the selected column as a column heading. This heading may not be descriptive and, therefore, maybe difficult to understand. Column alias can be used in such case. Column  alias  are specified after the column in the SELECT list using a space as a separator. By default, alias headings appear in uppercase. If the alias contains spaces or special characters (such as # or $), or if it is case-sensitive, enclose the alias in double quotation marks (" ").

Nulls in database
​Nulls have a special treatment in RDBMS.  Null value is actually 'no value' in a perticular column. Null is neither numeric nor character. It is just 'no value' and hence called as null. Inserting NULLs , deleting nulls and retriving nulls needs different methods.
Let us see the working of NULLs in the tables.
Defining a Null Value       
  •      If a row lacks a data value for a particular column, that value is said to be null or to contains  a null.
  •      A null is a value that is unavailable, unassigned, unknown, or inapplicable.
  •      A null is not the same as a zero or a space. Zero is a number, and a space is a character.
  •      Columns of any data type can contain nulls. However, some constraints (NOT NULL and PRIMARY KEY) are       used  to prevent nulls from  being used in the column.
Null Values in Arithmetic Expression
  •        If any column value in an arithmetic expression is null, the result is null.
  •        For example, if you attempt to perform division by zero, you get an error.
  •        However, if you divide a number by null, the result is a null or unknown.

Limit clause

Sometimes we need to restrict the rows while displaying. If only a specified number of rows are required  from a result set,   LIMIT clause  can  be used in the query.   Limit clause will display only specific number of rows.

Syntax:
SELECT column_name(s)            
FROM table_name
[WHERE condition
 GROUP BY column_name
 HAVING condition on aggregate function
 ORDER BY column_name 
 LIMIT number [OFFSET number];]

Limit clause will be the last clause in select statement. Limit clause can be used on ordered, indexed or grouped data as well. 
If multiple rows have identical values in the ORDER BY columns, the server is free to return those rows in any order, and may do so differently depending on the overall execution plan.

Case statement

Syntax:
CASE 
WHEN search_condition THEN statement_list 
[WHEN search_condition THEN statement_list] 
...    
[ELSE statement_list] 
END

Sorting data
Sorting of the data is a very basic user requirement.  Anytime whenever we think of a data, irrespective of the size, data sorting cannot be avoided. MySQL has made this process very easy by giving a ORDER BY CLAUSE.

Syntax :
SELECT column_name(s)
FROM table_name
[WHERE condition
 GROUP BY column_name
 HAVING condition on aggregate function
 ORDER BY column_name  ASC / DESC];

Data is ordered at the end. This will be the last statement executed in the select clause.
